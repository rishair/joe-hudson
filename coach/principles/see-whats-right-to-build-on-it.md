# See What's Right and Build On It

## The Principle

There are two paths to a solution. The first: see what is wrong, figure out how to fix it. The second: see what is right, figure out how to build on it. Most people and most organizations use only the first path, because the negativity bias of the human brain makes the first path easier to access. The second path is available but requires deliberate practice to see.

The principle is not that the second path replaces the first. The principle is that solutions require both. Using only the first path produces solutions made on incomplete data.

"There's kind of two ways to get to solutions. One way to get to solution is to see what's wrong and how do you fix it. And the other way to get to a solution is see what's right and how do you build on what's right." -- Joe Hudson, How Does Gratitude Create Better Teams?

## Why the First Path Dominates

Evolution. The brain was tuned for survival, and survival required catching threats. Missing the snake is fatal; missing the apple is not. The negativity bias is not a bug, it is an adaptation. It is also actively reinforced in modern environments where the consequences of negative outcomes are higher status (the post-mortem, the blame, the firing) than the consequences of positive outcomes (a quiet success).

"What's been really useful for humanity up until now is to be able to see what's wrong more than see what's right. See what's right is, 'Oh, I like that apple.' See what's wrong is, 'Oh, that snake can kill me.' And so we've learned, and there's all sorts of psychological studies that we tend to look for what's wrong." -- Joe Hudson, How Does Gratitude Create Better Teams?

This generalizes: the apparatus that scans for snakes will also scan for what is wrong in a relationship, in a team, in the self. The same mechanism that saved the species installs the inner critic that runs all day. See [[voice-in-the-head]].

Intelligence amplifies the bias, not corrects it. See [[intelligence-amplifies-self-criticism]].

## What the Second Path Requires

Seeing what is right requires interrupting the default. The interrupt is not "be more positive." That instruction tends to produce a forced positivity that the system rejects as inauthentic. The interrupt is structural: create a deliberate practice that forces positive data into the room.

Gratitude practices are one such structural interrupt. See [[gratitude-rounds-before-meetings]].

Looking for what is working in the situation that contains the problem is another. The Vietnam village where children were not malnourished -- the second path looks for that village. See [[vietnam-malnutrition-story]]. The first path would have spent the money distributing food.

The orientation extends inward. The person in a difficult emotional pattern can be looking for what is right inside the pattern -- not as a way to escape it, but as additional data. The pattern was trying to take care of something. See [[honor-the-patterns-intent]].

## The False Dichotomy It Replaces

The principle is not "look for positives instead of negatives." The principle is "look for both." A team that scans only for what is wrong has half the data. A team that scans only for what is right has the other half. Decisions made on either half alone are worse than decisions made on the whole.

"Because we're being positive, we're not being Pollyannish. I want to hear everything that's going wrong. I just also want to hear everything that's going right." -- Joe Hudson, How Does Gratitude Create Better Teams?

The positive scan is additive. It does not replace the negative scan. The leader who institutes gratitude rounds does not stop running incident reviews. The two are happening in parallel.

## The Application to Problems

When a specific problem is on the table, the second path looks like: where is this problem not happening? What about the problem is also an opportunity? What resources are revealed by the problem?

The non-selling product example: the customers are giving feedback (resource), the salespeople are surfacing objections (resource), the team is learning what makes products sell before building the next one (resource). The same situation that is a problem is also rich with assets. The first path sees only the deficit. The second path sees the assets that the deficit reveals.

This is not reframing as denial. The product still is not selling. The second path adds data; it does not subtract.

## The Application to Self

The same principle applies inward. A person in a difficult emotional state who scans only for what is wrong sees a long list of personal deficits. A person who scans for what is right -- what is functioning, what survived, what was learned -- sees a different self.

This is connected to but not identical with [[negative-bias-of-trauma-imprinting]]. That pattern is about what the brain encoded from a difficult past. This principle is about what to actively look at in the present. The two intersect when adult self-perception is filtered by childhood encoding that missed the positive evidence.

## The Practical Tests

A leader can check whether they are running both paths:

- After a meeting, can you list both what went wrong and what went right? If only one list comes easily, one path is running.
- In a one-on-one, are you investigating both the struggles and the wins? If only one, one path is running.
- In a strategy session, are you analyzing both the failures and the unexpected successes? If only one, one path is running.

The unexpected success is a frequent miss. Things that worked unexpectedly well rarely get the same investigation as things that failed unexpectedly. The information is asymmetrically extracted. The unexpected failure produces a post-mortem. The unexpected success produces a quiet celebration and then nothing.

## When Someone Experiences This

- A leader has been working hard on what is broken and feels exhausted with no traction
- A team is in negative-spiral mode where every meeting catalogs more problems
- A person doing self-work has been focused on what is wrong with them for a long time and has stopped seeing what is functioning
- An organization is trying to fix a system that has working examples within it that no one has studied

## Upstream (What Leads Here)

- A leader asks how to address team dysfunction without making people feel attacked
- A team is in a problem-finding rut and cannot generate solutions
- A person describes themselves entirely in terms of deficits
- A coaching session where the client only catalogs what is wrong
- A diagnostic phase that has been thorough on the problems and absent on the assets

## Tone

Matter-of-fact. The principle is presented as a structural observation about how brains work, not as a moral imperative to be positive. The negativity bias is named without contempt; it is just a thing that exists and has to be worked with.
