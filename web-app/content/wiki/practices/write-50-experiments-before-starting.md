# Write 50 Experiments Before Starting (The Pre-Built Iteration List)

## The Practice

Before attempting the thing you have been trying and failing to do, sit down and write 50 different experiments you could run on it. Write them all out. Number them. Then begin doing them in order.

The list is the structural reframe. With the list in front of you, no individual attempt can collapse into "I failed." There is only "I ran iteration #1; I learned X; now I run iteration #2." The failure category is structurally unavailable because the next iteration is always already on the page.

"Next time you want to lose weight, sit down and write 50 experiments you can run about losing weight. I'm going to do an Ayurvedic diet. I'm going to do a traditional Chinese medicine diet. I'm going to do the Atkins diet. I'm going to try this exercise, this exercise, this exercise. I'm gonna count calories. Oh, I'm gonna control my blood sugar. I'm gonna wear a blood sugar monitor. I'm gonna work with my hormones. Like, there are so many ideas of how to lose weight. Write down every experiment that you're gonna do and then do them in order." -- Joe Hudson, Failure

## Instructions

1. **Pick the thing you have been trying and failing at.** Losing weight. Finding a partner. Starting the business. Writing the book. Healing the relationship. The thing where prior attempts have collapsed into "I failed at this." Pick one.

2. **Sit down and write 50 different ways you could try it.** Not 5 ways. Not 10 ways. 50. The number matters because most people stop at 3 or 4 obvious attempts; the obvious attempts are usually the ones the prior failure verdicts attached to. Get past those. Write the weird ones. Write the ones you have not tried. Write the ones other people swear by. Write the ones from books, from podcasts, from friends, from random search results. The list itself is the work.

3. **Number them. Order them.** Pick an order. Any order. The order is less important than the existence of the order. Knowing what is next is what makes any individual iteration safe.

4. **Begin doing them in order.** Run iteration #1 with whatever time and resources are appropriate. Note what you learned. Check the box. Move to iteration #2.

5. **Treat each check as a win.** "Frame it so that every time you make a check on an experiment, it's a win. Like, oh yeah, look at that. I want -- like you're checking off a task list. You know, every one of those is a win." -- Joe Hudson.

6. **Feel the emotions on the way through.** Disappointment, sadness, frustration, fear -- all of it is part of the iteration. The list does not bypass the feelings. It gives the feelings a structural place to return into. Feel them, then move to the next item. See [[emotional-fluidity]].

## Purpose

The practice has three coordinated effects:

**Pre-empting the habenula.** The brain structure that suppresses retry after perceived failure ([[habenula]]) operates on the perception of failure. The pre-built list changes what the next event registers as. The same outcome that would have registered as failure registers instead as "iteration #1 complete; iteration #2 next." The retry happens because the brain did not get the suppression signal.

**Defusing the verdict frame.** The verdict frame requires a finish line. The list has no finish line until item #50 is checked. The verdict cannot land on any single iteration because the iteration was always part of a longer sequence. Even item #50, if the goal is not reached, is just the prompt to write the next list of 50.

**Converting the should into a curiosity.** The "I should lose weight" framing produces resistance because shoulding never works ([[rebellion-or-submission-to-should]]). The list converts the should into a series of curiosity-driven experiments. "I want to see what happens with the Ayurvedic diet." "I want to see what happens with blood sugar monitoring." Each item is a question rather than an obligation.

## What It Tends to Produce

The most common report from clients who do the practice:

- **Reduced shame on each iteration.** Each result registers as data rather than verdict.
- **More willingness to try the weird ones.** The list contains items the person would not have tried as standalone attempts. Inside the list structure, the weird ones become normal.
- **Surprising learning from the items that "didn't work."** Each iteration produces information the person did not have before. The information accumulates. The eventual breakthrough often comes from synthesis of multiple iterations rather than from any single iteration being the right one.
- **The thing they have been trying for years starts to move.** Not because any single item is magic, but because they are finally iterating instead of failing.
- **The iterative orientation generalizes.** Once the structure has been built for one domain, it tends to spread. The person starts treating other long-stuck areas the same way.

- When introducing the iterative mindset to a person who responds well to concrete protocols.

## What to Watch For

- **The person who writes 5 items and stops.** The number matters. 50 is uncomfortable. The discomfort is partly the point: the list-writing itself is the first iteration of breaking the failure-verdict frame. If they cannot write 50, the inquiry becomes: what is in the way of generating 50? Often it is the same constriction that has been preventing iteration all along.
- **The person who writes the list and then waits for the "best" item to start with.** The order matters less than the doing. The premise of waiting for the best item is the achievement-frame returning. The instruction is to pick any order and start.
- **The person who runs iteration #1 with everything riding on it.** The list has been built but the verdict-frame is still operating. Reaffirm: iteration #1 is iteration #1. The whole list is the sequence. No single item carries the weight.
- **The person who skips the feeling on the way through.** The list is a structural protection, not an emotional bypass. If the person is using the list to compartmentalize the disappointment of each iteration, the learning will be intellectual without being emotional. See the emotion-on-the-way-through point in [[iterative-mindset]].

## Variations

For partner-seeking: "I'm going to try this app for dating. This app for dating. Go to yoga classes." Joe explicitly extends the practice to the dating domain in the transcript.

For the artist or salesperson: a variant is the [[hairdresser-1000-for-50-rejections]] move -- the list becomes a quota of rejections to collect, with a reward attached. The reframe is identical: each rejection is a check, not a failure.

For a company introducing iteration culture: the list can be team-generated. Every team member contributes 5 experiments to a shared list. The team works the list together. See [[reward-failure-in-the-organization]] and [[team-as-scientific-method]].

## Tone

Joe delivers the practice with the energy of a structural fix. The voice is concrete, practical, slightly impatient with the alternative ("we say, oh, we're going to do this diet. We're going to try. X amount of time goes by. And then we fail in some way and we give up"). The practice is offered as the obvious operational form of a structural insight, not as a self-help technique.
