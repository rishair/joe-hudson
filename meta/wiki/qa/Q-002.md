# QA: Coach-version selector: switching variants, behavior differs, persistence tags variant, deploy live

## Scope

The in-app coach-version selector from [[E-054]]: the control renders, switching it changes which coach variant answers, the four variants produce stance-distinct replies, and the existing surfaces (streaming, attribution modal, persistence, cost-cap) do not regress.

## Test plan

Golden path first, then edge cases.

1. **Selector renders.** Load the deployed app. The coach-variant selector is visible and lists four options (`v5b`, `reflective`, `grounded`, `joe-voiced` or their labels), defaulting to `v5b`.
2. **Behavior differs by variant.** Send the same opening message ("I have a feeling I don't know how to let people in") under each of the four variants in separate conversations. Confirm: `v5b` ends in a bare curious question with no declarative substance; `reflective`/`grounded`/`joe-voiced` each share a read/frame/observation as a statement. Capture the four replies as evidence.
3. **Switch takes effect.** Confirm the selected variant actually drives the reply (not just a cosmetic dropdown) — e.g. network request body carries the profile id, or the reply stance matches the selection.
4. **No regression.** For at least one variant: streaming renders progressively, the "N sources" attribution modal opens and shows resources, the conversation persists across reload (SQLite/OPFS), and the cost-cap still guards.
5. **Persistence tagging (if E-054 shipped it).** A saved conversation records which variant produced each assistant message; verify after reload.

### Edge cases
- Switching variant mid-conversation: the next turn uses the new variant, prior turns unaffected.
- Unknown/missing profile id in request: server falls back to `v5b`, no error.
- Mobile viewport: selector is reachable and usable (not clipped).

## Pre-conditions

- [[E-054]] deployed live to the Cloudflare URL (currently `https://joe-hudson-coach.rishi-ishairzay.workers.dev`; confirm the redeploy landed).
- `OPENROUTER_API_KEY` funded (real LLM calls happen during the test).
- No auth required (public app).

Drive the deployed URL, screenshot each variant's reply, capture console errors and the chat request body. Run desktop and one mobile viewport.
