# Build eval suite for Joe Hudson AI coach

## Outcome

A runnable evaluation framework that scores any AI chat system on how faithfully and effectively it coaches like Joe Hudson. The eval produces a structured scorecard across multiple dimensions: methodology fidelity, tone, coaching effectiveness, safety, and anti-pattern avoidance. The eval uses LLM-as-judge with rubrics derived directly from the `coach/` compendium, simulated client profiles derived from `coach/concerns/`, and multi-turn conversation testing. The output is a repeatable, automated test suite that can be run against any candidate system prompt or retrieval configuration, producing scores that guide iteration.

## Correctness

### Must-haves

- **Eval rubrics** exist as structured files covering at least 5 dimensions: (1) AoA methodology fidelity, (2) coaching tone, (3) coaching effectiveness/depth, (4) safety, (5) anti-pattern avoidance
- Each rubric dimension has a 1-3 scoring scale with concrete behavioral anchors (what a 1, 2, and 3 look like), not vague descriptions
- **Simulated client profiles** (minimum 15) derived from `coach/concerns/` directory. Each profile specifies: the concern, opening statement(s), emotional state, resistance patterns, and what correct coaching looks like (expected moves, reads, concepts)
- **Scenario types** cover at least: (1) happy path (open client), (2) resistance (client intellectualizes or deflects), (3) crisis/safety (self-harm, abuse), (4) edge cases (client asks coach to break methodology), (5) red herring (surface issue masks deeper pattern)
- **Multi-turn test harness** that runs a simulated client against the coach-under-test for 8-15 turns, then scores the conversation with LLM-as-judge
- **Anti-pattern test cases** derived from `coach/anti-patterns/` — scenarios specifically designed to trigger each anti-pattern, with scoring that detects whether the coach fell into it
- **Methodology fidelity behavioral checklist** that checks for: use of signature questions (from `coach/questions/`), correct reads (from `coach/reads/`), appropriate moves (from `coach/moves/`), correct framework application (from `coach/frameworks/`), and adherence to principles (from `coach/principles/`)
- **Safety hard-fails** — binary pass/fail checks that override all other scores: failure to recognize crisis, providing medical/psychiatric advice beyond scope, reinforcing harmful beliefs, breaking character with generic LLM disclaimers mid-session
- **Per-turn annotations** in addition to per-conversation scores — the judge identifies specific turns where coaching quality shifted, enabling targeted iteration on the coach prompt (a score of "2" tells you something is mediocre; per-turn annotations tell you *where* it went wrong)
- **Judge calibration** — the judge has been validated against gold-standard conversations (known-good, known-bad, real Joe coaching) and produces consistent scores across repeated runs before being trusted for the full eval
- **Ground-truth positive controls** — at least 3 real Joe Hudson coaching exchanges from `transcripts/` are used as positive controls that the eval must score highly; if the eval penalizes actual Joe coaching, the rubrics are wrong
- **Few-shot calibration examples** for the judge, drawn from actual Joe Hudson transcripts — these ground the judge's scoring in real examples rather than abstract rubric descriptions
- Eval output is a structured JSON scorecard per conversation (with per-turn annotations), plus aggregate scores across the full test suite
- Eval is runnable via a single command (script or CLI)
- All rubrics, profiles, and scoring criteria reference specific `coach/` files by ID so they stay grounded in the compendium
- **Cost and runtime documented** — estimated API calls, tokens, wall-clock time, and dollar cost for a full eval run, so iteration speed is understood

### Nice-to-haves

- Pairwise comparison mode: run the same profile against two coach configurations and pick a winner per dimension
- Regression tracking: store historical scores to detect when a change improves or degrades performance
- Coverage report: which `coach/` files were exercised by the test suite and which were not
- Human eval integration: a mode that presents conversations to a human rater alongside the rubric
- Multi-judge aggregation: run 2-3 judge models and take consensus to reduce individual model bias

## Exit Criteria

- [ ] At least 5 rubric dimension files exist with 1-3 scoring scales and behavioral anchors
- [ ] At least 15 simulated client profiles exist, covering all 5 scenario types (with >= 3 crisis profiles)
- [ ] At least 10 anti-pattern test cases exist, each targeting a specific `coach/anti-patterns/` entry
- [ ] A methodology fidelity checklist references at least 10 questions, 10 reads, 10 moves, with appropriateness AND inappropriateness conditions and negative examples
- [ ] Safety hard-fail criteria are defined with at least 6 failure modes, distinct from anti-patterns
- [ ] A runnable test harness exists that executes multi-turn conversations and produces JSON scorecards with per-turn annotations
- [ ] At least 3 gold-standard real Joe Hudson coaching exchanges extracted from transcripts and used as positive controls
- [ ] Judge calibration has been performed: gold conversations score in expected ranges, self-consistency >= 80%
- [ ] Simulated client LLM behavior has been validated (character consistency, resistance fidelity, reproducibility)
- [ ] The eval has been run at least once against a baseline coach system prompt and produced interpretable results
- [ ] Judge model selection is documented with rationale
- [ ] Eval cost and runtime are documented
- [ ] All eval components are documented enough that a new agent can run the eval and interpret results

## Context

G-007 built a coaching compendium at `coach/` with ~470 files across 14 categories: concepts, concerns, moves, reads, patterns, distinctions, principles, questions, practices, frameworks, anti-patterns, examples, applications, and arcs. This compendium is the knowledge base for a Joe Hudson AI coach.

Before building the coach itself, we need an eval that tells us whether a given coach implementation is good. Without an eval, we are flying blind — making prompt changes and guessing whether they helped. The eval is the feedback loop that makes the coach buildable.

The eval design draws on:
- **TherapyGym** (Stanford, 2026): Clinical fidelity scoring via adapted CTRS, safety annotation, simulated patients
- **MITI coding system**: The gold standard for scoring adherence to a specific coaching methodology (Motivational Interviewing). We adapt this approach for Art of Accomplishment
- **CharacterBench / PersonaGym**: Persona fidelity evaluation across multiple dimensions
- **MindEval**: Multi-turn mental health conversation evaluation with patient LLM + judge LLM
- **DeepEval**: Open-source Python framework with native multi-turn eval support

The `coach/` compendium provides unique advantages for eval design:
- `concerns/` map directly to simulated client profiles (what someone walks in saying)
- `anti-patterns/` are a ready-made list of failure modes to test for
- `reads/` define what the coach should *notice* — testable via judge rubric
- `moves/` define what the coach should *do* — testable via behavioral checklist
- `questions/` are signature utterances that indicate methodology fidelity
- The `related` links in each file encode the correct coaching navigation path
