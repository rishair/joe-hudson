# Redesign coach web app with a warm, somatic aesthetic

## Outcome

The coach web app is visually redesigned so it reads as warm, calm, grounded, and human, the opposite of a clinical SaaS dashboard or a generic chatbot. A person opening it feels they can slow down and be honest. The redesign follows the [[frontend-aesthetics]] playbook, uses the committed warm-somatic theme, works on mobile, and houses the coach-variant selector from [[G-014]] in the new shell.

## Correctness

The current app is functional but unstyled (default-ish shell, the AI-slop baseline the aesthetics playbook warns against), and [[Q-001]] found two mobile-responsiveness defects (sidebar never collapses; ~527px horizontal overflow at a 412px viewport). The redesign fixes both while committing to one cohesive aesthetic.

**Committed theme (locked, chosen by Rishi 2026-05-25): warm & somatic.**
- **Typography:** Fraunces (display/headings, a soft humanist serif) paired with a humanist sans for body (a non-default choice such as a warm grotesque or IBM Plex Sans — NOT Inter/Roboto/Arial/system). High weight contrast (light body vs heavy display), large type-scale jumps.
- **Color:** warm sand/cream background, deep ink/espresso text, one grounding terracotta-ochre accent. Dominant warm neutral + single sharp accent, not an evenly-spread palette. No corporate blue, no AI-purple-on-white. All colors as CSS variables (design tokens).
- **Space:** generous whitespace; unhurried, editorial rhythm. The chat should breathe, not feel like a form.
- **Motion:** quiet. One well-orchestrated page-load with a soft staggered fade (`animation-delay`), gentle message-in transitions. No scattered flashy micro-interactions.
- **Background:** subtle warmth/depth (a soft gradient or paper texture), not a flat fill.

### Must-haves

- The deployed app uses the warm-somatic theme: Fraunces (or equally distinctive non-default display) + a non-default body font, warm-neutral background, deep ink text, single warm accent, all via CSS variables.
- No Inter/Roboto/Arial/system-font as a primary face; no purple-gradient-on-white; no flat default shell.
- Mobile works: the [[Q-001]] defects are fixed (sidebar collapses or adapts at small viewports; no horizontal overflow at 360-414px widths; chat input usable).
- All existing functionality survives: chat + streaming ([[E-052]]), resource-attribution modal ([[E-044]]), persistence ([[E-041]]), welcome flow ([[E-045]]), wiki view ([[E-042]]), and the coach-variant selector ([[E-054]]) are restyled but fully working.
- Deployed live to the Cloudflare URL.
- [[Q-003]] QA passes on desktop and mobile.

### Nice-to-haves

- A theme-constraint prompt block (per the playbook's "lock a theme" pattern) saved alongside the code so future generations stay on-aesthetic.
- Dark-mode variant of the warm theme (warm charcoal / cream / amber) toggleable.
- Tasteful page-load orchestration on first visit.

## Exit Criteria

- [ ] App uses the warm-somatic theme (distinctive non-default fonts, warm palette via CSS variables) — verifiable in the deployed UI
- [ ] No primary use of Inter/Roboto/Arial/system fonts; no purple-on-white
- [ ] [[Q-001]] mobile defects fixed (no horizontal overflow at 412px; sidebar adapts)
- [ ] All prior surfaces (chat/streaming/modal/persistence/welcome/wiki/selector) work post-redesign (no regression)
- [ ] Redesign deployed live to Cloudflare
- [ ] [[Q-003]] verdict is `passed` (visual fidelity + no functional regression, desktop and mobile)
