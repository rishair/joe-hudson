# Repeatable idempotent sync system for all content

## Outcome

A single script (e.g., `./sync.sh`) that, when run:
1. Checks all known content sources (YouTube channel, podcast feed, guest appearances) for new items since the last run
2. Downloads any new items as audio files into the `transcripts/` directory structure
3. Transcribes new items with speaker diarization and voiceprint-based Joe Hudson labeling
4. No-ops cleanly when there is nothing new (no errors, no re-downloads, no re-transcriptions)
5. Can be run on a cron schedule or manually at any time

The system should be smart about detecting "new": it compares against what already exists on disk, not against a hardcoded list. It should handle partial failures gracefully (resume where it left off).

## Exit Criteria

- [ ] A `sync.sh` script exists at the project root
- [ ] Running `sync.sh` when nothing is new produces no downloads and no errors
- [ ] Running `sync.sh` after a new YouTube video is published downloads only that video
- [ ] Running `sync.sh` after a new podcast episode is published downloads only that episode
- [ ] New downloads are automatically transcribed with speaker labels
- [ ] The script is documented with usage instructions in a README or inline help
- [ ] The script has been tested end-to-end at least once on a real new item

## Context

The current download and transcription pipelines were built as one-shot experiments. They work but require manual invocation with specific commands. This goal turns them into a production-ready system that can run unattended.

Key design inputs from completed experiments:
- [[E-008]]/[[E-010]]: yt-dlp with `--download-archive archive.txt` already handles YouTube idempotency
- [[E-009]]/[[E-011]]: Podcast download uses skip-if-exists logic on directory names
- [[E-013]]: Joe Hudson voiceprint is saved at `meta/raw/joe-hudson-voiceprint.npy`
- [[E-005]]: WhisperX transcription pipeline (pending completion, will inform transcription approach)

The script should compose these proven pipelines into a single repeatable flow.
