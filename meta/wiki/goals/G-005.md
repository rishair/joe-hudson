# Execute full bulk download of all content

## Outcome

All identified content is downloaded locally as audio files in the `transcripts/` directory structure:
- All 329 YouTube videos downloaded as MP3 audio (via yt-dlp)
- All 167 podcast episodes downloaded as MP3 audio (via iTunes API URLs)
- Each file in its own directory: `transcripts/{YYYY-MM-DD}_{title}/{title}.mp3`
- A verification report confirming counts match expected totals and no corrupted files

## Context

Both download pipelines have been validated by experiments:
- [[E-008]] proved yt-dlp works for YouTube (5/5 test downloads, no cookies needed, 96kbps)
- [[E-009]] proved curl with iTunes API URLs works for podcast (3/3 test downloads, 128kbps)

This goal is the execution phase: running the full downloads end-to-end. The pipelines are proven; this is about running them at scale and verifying completeness. Downloads can run in parallel (YouTube and podcast are independent sources).

**Estimated totals**: ~5.8 GB YouTube + ~7.8 GB podcast = ~13.6 GB total. Runtime: 45-90 minutes combined.

**Merge strategy**: Both sources write to the same `transcripts/` directory with date-prefixed folders. The `archive.txt` file prevents YouTube re-downloads. Podcast downloads skip existing files by checking directory existence.
