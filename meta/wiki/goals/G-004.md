# Transcribe all media with speaker diarization

## Outcome

Every downloaded media file has a `transcript.md` in its directory with:
- Full text transcript
- Speaker labels (at minimum: "Joe Hudson" vs "Speaker 2", ideally with guest names when known from MEDIA.md)
- Timestamps at reasonable intervals (per paragraph or per speaker turn)

The transcripts should be accurate enough to be useful as a knowledge base. Diarization should correctly attribute Joe Hudson's speech at least 90% of the time.

## Context

Depends on [[G-003]] having media files downloaded. Can start transcribing as files become available. The key technical challenge is speaker diarization: distinguishing Joe Hudson's voice from guests. Most episodes are 2-person conversations (Joe + one guest), which is the easier case. Some may be solo or multi-guest.

### Budget Constraint

An OpenRouter API key is available (stored in `.env` as `OPENROUTER_API_KEY`) with $20 credit. Quality of diarization output is the priority. Strategy: if the chosen approach costs money per episode, build a prototype first that proves the pipeline works on 2-3 episodes before running it on the full catalog. Do not burn the $20 on a bulk run until the prototype demonstrates acceptable quality. Free/local tools are preferred if they match API quality.
