# Tune coach authority via in-app version selector and authority-spectrum variants

## Outcome

Rishi can switch between multiple coach variants inside the deployed web app and vibe-check them back to back. Through that loop, the team identifies the prompt-stance calibration that makes the text coach feel like it has authority (it shares substance, names what it notices, offers frames) without tipping into therapist-interpretation or advice-giving.

## Correctness

The vibe-check of v5b (REQ-001, 2026-05-25) found the coach lands on presence and curiosity but reads as hollow: it only asks open curious questions and never offers a read, frame, or named observation as a statement, so the user did not trust it. In live sessions Joe's authority is carried by tone, posture, presence, and timing, all of which text strips away. The hypothesis under test: a text coach must compensate by sharing more substance to earn trust, and there is a calibration band between "too passive" (v5b) and "overdoing it" (advice/interpretation, which the methodology forbids).

### Must-haves

- The deployed web app has a visible control to select which coach variant a conversation uses. The selection takes effect for that conversation's replies.
- Four selectable variants exist, all sharing the v5b guided-walk retrieval substrate (Sonnet walker, same K/step budget) so the system prompt is the only variable: `v5b` (baseline control), `reflective`, `grounded`, `joe-voiced`. See [[E-053]] for stance definitions.
- Each variant is a versioned prompt file in the web app, traceable to a definition (no inline magic strings).
- Switching variants produces observably different coach behavior on the same input (a reflective turn reads differently from a baseline turn).
- The change is deployed live at the Cloudflare URL, not just local.
- A QA pass ([[Q-002]]) confirms the selector works end to end and persistence records which variant produced each message.

### Nice-to-haves

- The variant in use is recorded per-message in the SQLite persistence layer, so a saved conversation shows which coach answered.
- The selector label explains each variant in one phrase (hover or subtitle).
- A second vibe-check session (REQ-001 session 2) is run by Rishi across the variants, with a chosen winner or a documented "none land, here's why."
- Optional smoke eval (2-profile) on the chosen winner before it is promoted to default, to catch safety/quality regression from higher-authority prompts.

## Exit Criteria

- [ ] Web app exposes a coach-variant selector control (verifiable in the deployed UI)
- [ ] Four variant prompt files exist under the web app, one per stance, all on the v5b retrieval substrate
- [ ] Selecting a variant routes the chat request to that variant's system prompt server-side ([[E-054]] succeeded)
- [ ] [[Q-002]] verdict is `passed` (selector switches variants, behavior differs, deploy live)
- [ ] Rishi has run a second vibe-check across variants and recorded impressions on [[REQ-001]]

## Context

This goal is the continuation of [[G-009]] (build the coach) driven by the [[REQ-001]] vibe-check, using the [[G-010]] web app as the test vehicle. G-009 met its numeric eval bars (overall 2.71, 15/15 safety) but the qualitative vibe-check found a felt-authority gap the rubric missed. The G-009 iteration log already flagged the prompt layer as the untested lever (E-035 showed the catastrophic edge-not-anxious regression traced to coach interpretation reflex, not retrieval). This goal gives that prompt-layer work a concrete direction and a fast human feedback loop. G-009 does not close until this resolves.

Three implementation choices worth recording: (1) the `v5b` control imports the deployed `SYSTEM_PROMPT` directly (byte-identical, no drift); (2) the three new variants are `BASE_PROMPT + stance-layer` compositions rather than four independent copies, so SAFETY FIRST + the anti-pattern guardrails are intact in all four by construction (a faithful read of "differ only in the stance layer"); (3) `resolveCoachProfile()` falls back to `v5b` on any unknown/missing id, so E-054's route degrades gracefully. The authority spectrum is monotonic in declarative substance volunteered (questions-only → +read → +frame → +challenge), which is what lets the REQ-001 session-2 vibe-check *locate a band* rather than just rank. Variant example phrasings are deliberately written around the session-1 concern (glimmer/birthday) for legible contrast.
