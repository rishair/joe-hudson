# In-line lesson surfacing during a user session

## The idea

The `coach/` compendium has hundreds of Joe Hudson's "lessons" — concepts, distinctions, principles, frameworks, anti-pattern descriptions. They're knowledge ABOUT coaching, not just material for the coach to use. There's a question worth thinking through: when (if ever) during a user's session would surfacing one of these directly to the user — as a teaching moment, not as something the coach embeds in its response — actually serve them?

Examples of what this might look like (none are decided):
- After a session ends, "you might find this useful: [[loving-to-transform]] — Joe's note on the pattern this conversation touched"
- Inline-as-aside ("Joe calls this 'the rush signature' if you want to read more")
- A "lessons from this conversation" digest accessible from the conversation history
- A separate "explore" mode where users browse the compendium guided by what their conversations have touched on

## Open questions

- **When in the user lifecycle does this fit?** Mid-session is intrusive (breaks the coaching flow per R-017's "subtle" principle). End-of-session feels right but only if the user wants reflection material. First-time-user wants to read about the methodology — but that's the welcome flow.
- **What's the trigger?** Pattern-matching the conversation against `coach/` themes (similar to retrieval but for surfacing to the USER, not the coach)? Or user-initiated ("show me what concepts came up")?
- **Tone**: educational vs. invitational? "You're experiencing X" vs. "here's what Joe says about X if you're curious."
- **Does this conflict with the unaffiliation framing?** The welcome flow explicitly says "we draw on Joe's publicly available content." Surfacing his lessons by name leans into that — fine if framed right.

## Why this is backlog (not a current goal)

It's a real product question that needs user research more than engineering. Live the v1 first — see how people actually use the coach, when (if ever) they ask about the underlying methodology themselves. Then decide whether to add this proactively or wait for users to pull it.
