# OpenRouter API key needed for G-010 web app

## What's needed

An OpenRouter API key, set in `.env` as `OPENROUTER_API_KEY=...`. OpenRouter accounts are free to create at https://openrouter.ai; pre-funding the account with $5-10 is enough to develop and test G-010 locally.

## Why it's needed

The [[ai-web-app]] playbook mandates OpenRouter as the LLM gateway for this stack. Reasoning (per the playbook): one key, one billing source, easy model switching, model-agnostic — at a small cost premium that's worth the operational simplicity. The G-010 chat surface must call LLMs through this gateway, not direct to providers, so the eventual config-switching ("try this conversation with the Haiku walker instead") is a single env-var change rather than a code rewrite.

## What's blocked without it

- E-040 (scaffold minimal chat) cannot validate end-to-end without an actual LLM call
- E-043 (wire v5b coach into route handler) is fully blocked
- Any other LLM-calling experiment under G-010

Workaround if the user wants to defer the key: experiments R-015, R-016, R-017, R-018 and the foundation experiment E-040 (ingestion script) do not need any LLM calls and can run first. The blocker only fires when the app needs to actually talk to a model.

## Notes

The existing `ANTHROPIC_API_KEY` in `.env` could technically be used as a direct-to-Anthropic fallback if OpenRouter integration is blocked, but that would diverge from the playbook and complicate later model swapping. Not recommended unless OpenRouter is unavailable for some reason.

## Resolution

Resolved 2026-05-24. `OPENROUTER_API_KEY` added to `.env` (gitignored). User also provided `HF_TOKEN` (HuggingFace) in the same message; stashed alongside for future use by experiments that need HuggingFace model access. E-040 and E-043 are unblocked on credentials.
