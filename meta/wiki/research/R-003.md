# How to bulk-download YouTube channels and podcast feeds in 2026

## Question

What tools and methods currently work for bulk-downloading all videos from a YouTube channel and all episodes from a podcast RSS feed? Have yt-dlp or similar tools been affected by recent YouTube changes?

## Findings

### YouTube: yt-dlp Status (May 2026)

yt-dlp remains the standard tool for YouTube downloads. It is functional but faces increased friction from YouTube in 2026:

- YouTube has "weaponized" SABR (Server-Based Adaptive Bit Rate) and aggressively kills connections from download tools.
- Users report HTTP 403 errors and connection resets, particularly without cookies.
- YouTube is phasing out the `android_sdkless` client that downloaders relied on.
- The nightly release channel is recommended over stable, as fixes for YouTube blocks ship faster.

**Mitigations that work:**
- Export browser cookies (via "Get cookies.txt locally" browser extension) and pass to yt-dlp with `--cookies cookies.txt`. This is the most reliable fix.
- Use `--sleep-interval 5 --max-sleep-interval 30` to add random delays between downloads.
- Use `--download-archive archive.txt` to skip already-downloaded files on re-runs.
- FFmpeg is required for audio extraction and format merging.

**yt-dlp is not installed locally.** It needs to be installed (via `pip install yt-dlp` or `brew install yt-dlp`).

**Audio-only extraction**: `yt-dlp -x --audio-format mp3 --embed-metadata` extracts audio and converts to MP3. Since [[G-004]] only needs audio for transcription, downloading audio-only saves storage and bandwidth.

**Full channel command**:
```
yt-dlp -x --audio-format mp3 --embed-metadata \
  --output "%(upload_date)s_%(title)s/%(title)s.%(ext)s" \
  --download-archive archive.txt \
  --cookies cookies.txt \
  --sleep-interval 5 --max-sleep-interval 30 \
  "https://www.youtube.com/@ArtofAccomplishment/videos"
```

### Podcast RSS: Simplecast Feed

[[R-001]] found the RSS feed at `https://feeds.simplecast.com/zksImfUP` but noted it only exposed ~25 recent episodes. Simplecast documentation confirms the RSS feed limit is configurable by the podcast owner (1 to 2000 episodes). The current ~25-episode window is a setting, not a hard platform limit. We cannot change it (we are not the podcast owner).

**Workarounds for the limited feed window:**

1. **PodcastIndex.org API**: The free API endpoint `/episodes/byfeedurl` returns all episodes ever indexed for a feed, not just those currently in the RSS XML. This likely contains the full history of ~160 episodes. Requires a free developer account.

2. **Official website scraping**: The Art of Accomplishment podcast page (https://www.artofaccomplishment.com/podcast) may list all episodes with direct audio links. This needs testing.

3. **Apple Podcasts / Spotify**: These platforms mirror the RSS feed. They do not provide bulk download APIs. Apple Podcasts shows episode listings that may contain a more complete history, but audio files are not directly downloadable in bulk.

**Podcast download tools:**

- **poddl** (Go, cross-platform): Downloads all/range of episodes from an RSS feed URL. Simple CLI.
- **rss-podcast-downloader** (Python): Downloads from RSS, writes ID3 metadata, tracks state in SQLite. More robust for large archives.
- **Simple curl/wget loop**: Podcast RSS feeds contain direct `<enclosure url="...">` links to MP3 files. A script can parse the XML and download each URL. This is trivial for the episodes that are in the feed.

For the episodes currently in the RSS feed (~25 most recent), any of these tools work directly. For the full archive, the PodcastIndex API or website scraping is needed first to get the audio URLs.

### Guest Appearances

[[R-001]] identified 10+ guest appearances on other podcasts. These are scattered across different channels:
- Most are available on YouTube (downloadable via yt-dlp with the specific video URL).
- Some are on podcast feeds of other shows (downloadable from those RSS feeds).
- Each requires individual URL lookup. There is no single feed.

## Implications

1. **Two parallel download tracks**: YouTube channel (yt-dlp) and podcast episodes (RSS + PodcastIndex). These can run independently.

2. **YouTube download is feasible but needs cookies**: The experiment should include exporting YouTube cookies from a logged-in browser session. Without cookies, 403 errors are likely.

3. **Podcast full archive requires PodcastIndex**: The RSS feed only has ~25 episodes. Getting all ~160 requires querying PodcastIndex.org or scraping the official website. This is the key blocker for podcast downloads.

4. **Audio-only is sufficient**: Since the end goal is transcription ([[G-004]]), downloading audio-only from YouTube saves significant storage. Video is not needed.

5. **Guest appearances are a separate, manual effort**: These should be a separate sub-goal or experiment, not mixed into the bulk download pipeline.

Recommended experiments:
1. Install yt-dlp, export cookies, test audio-only download of 3-5 YouTube videos from the AoA channel.
2. Query PodcastIndex API for the full episode list, then download all podcast audio files.
3. Separately catalog and download guest appearances.
