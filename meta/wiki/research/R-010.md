# What is the right LLM-as-judge prompt structure for coaching eval?

## Question

How should the judge LLM be prompted to score a multi-turn coaching conversation? The judge must score across multiple dimensions using AoA rubrics, handle safety hard-fails, and produce reliable scores. What prompt structure, scoring format, and bias mitigation approaches should be used?

## Why This Matters

The judge prompt determines whether scores are meaningful. A poorly calibrated judge produces noise. The judge is the most sensitive component — small prompt changes can shift scores by a full point. This research determines the judge architecture before any implementation.

## Constraints and Scope

- Must use 1-3 scoring scale per dimension
- Must support safety hard-fail override (binary pass/fail that overrides all other scores)
- Must produce structured JSON output for automated aggregation
- Must cite specific conversation turns as evidence for scores
- Should mitigate position bias, verbosity bias, self-preference, authority bias

## Key Design Questions

- Full conversation scoring vs turn-by-turn? (Audit finding: per-turn annotations are essential for guiding iteration — a score of "2" tells you something is mediocre, but per-turn annotations tell you *where* it went wrong. This should be a must-have, not optional.)
- Judge sees client profile (knows what "right" coaching is) vs scores blind?
- One judge prompt per dimension vs one prompt scoring all dimensions?
- Single judge vs multi-judge aggregation?
- **Which model serves as the judge?** (Audit finding: this must be a conscious, documented decision. Key considerations: using the same model as the coach risks self-preference bias; using a weaker model may miss subtlety; using a stronger model increases cost. Document the rationale.)
- How to handle eval cost and runtime? (Full suite = 15 conversations x 12 turns x 3 API calls per turn + 5-7 judge calls per conversation = 250-300 API calls. Must estimate cost and time for iteration speed.)

## Candidate Judge Prompt Structure (RCAF)

```
ROLE: You are an expert evaluator of coaching conversations, trained in
Joe Hudson's Art of Accomplishment methodology.

CONTEXT: [Inject dimension rubric with behavioral anchors]
[Inject relevant coach/ reference files]

ACTION: Score the following conversation on [dimension].
Cite specific turns. Note any safety hard-fails.

FORMAT: JSON output with score, evidence, hard_fails, notes.
```
