# Current state of speech-to-text with speaker diarization tools

## Question

What are the best current tools for transcribing audio with speaker diarization? What are the tradeoffs between local vs API-based solutions in terms of accuracy, cost, and speed for podcast-length content (30-90 minutes)? Can any tool label a known speaker (Joe Hudson) by name using a voice sample?

## Findings

### Local/Open-Source Tools

**WhisperX** (faster-whisper + pyannote 3.1 + wav2vec2 alignment) is the strongest local option for combined transcription and diarization. It bundles three components: faster-whisper for ASR, pyannote 3.1 for diarization, and wav2vec2 for word-level timestamp alignment. It runs 4x faster than base Whisper. GPU requirement: 5-6 GB VRAM total across all stages (pyannote alone needs 1.5 GB). Both Whisper and pyannote are free; costs are compute only. On a local Mac with Apple Silicon or a modest GPU, processing is feasible for 100+ episodes with no per-episode cost.

**Pyannote 3.1** is the current best open-source diarization model. Diarization Error Rate (DER) of 11-19% on standard benchmarks. For 2-speaker podcast conversations with clear audio, accuracy is at the high end of that range.

**faster-whisper** uses CTranslate2 to run Whisper models efficiently. The large-v3 model provides the best English accuracy. Word Error Rate (WER) for Whisper large-v3 on English podcast audio is competitive with commercial APIs.

### Commercial APIs

**AssemblyAI**: Base transcription at $0.15/hr (Universal-2) or $0.21/hr (Universal-3 Pro). Speaker diarization adds $0.02/hr. Total with diarization: ~$0.17-0.23/hr. For 100 episodes averaging 60 min each, cost is $17-23. Fits within the $20 budget but barely. DER improvements of 10% over their 2024 models; 30% improvement in noisy conditions.

**Deepgram Nova-3**: $0.46/hr base, diarization adds $0.12/hr. Total: ~$0.58/hr. For 100 episodes at 60 min: $58. Exceeds budget. Strongest advantage is noisy/phone-quality audio (2-4 WER points better than Whisper in those conditions). Podcast audio is typically clean, which reduces this advantage.

**Speechmatics**: Supports voice enrollment for known speaker identification. Accepts audio samples of specific speakers and labels them by name in output. This is the only commercial API found that directly supports the "label Joe Hudson" requirement out of the box.

### Known Speaker Identification

This was a key sub-question. Three approaches exist:

1. **Pyannote voiceprints (local, free)**: Pyannote supports speaker identification via voice embeddings. The workflow: extract an embedding from a sample of Joe Hudson's voice, then compare against diarized segments using cosine similarity. The `pyannote/embedding` model generates speaker embeddings. A GitHub project (z3lx/speaker-identification) implements this exact pipeline. This requires a short audio clip (~30-60 seconds) of Joe speaking alone.

2. **Speechmatics API (paid)**: Supports enrollment of known speakers via their API. Send a voice sample, get a voiceprint, and future transcriptions label that speaker by name. Pricing not confirmed but likely comparable to AssemblyAI.

3. **Post-hoc heuristic (free, no voice sample needed)**: In a 2-person podcast where Joe is always the host, the speaker who talks in every episode is Joe. Diarization assigns consistent labels per episode (Speaker A, Speaker B). A simple heuristic -- Joe is the speaker present in the first 30 seconds of every episode (he opens each one) -- could label him without any voice enrollment. This is less robust but zero-cost and zero-setup.

### Speed Estimates (Local)

WhisperX with large-v3 on Apple M-series (MPS backend) or a GPU with 6+ GB VRAM processes roughly 1 hour of audio in 5-15 minutes depending on hardware. For 100 episodes at 60 min average: 8-25 hours of compute. This can run unattended.

## Implications

The local approach (WhisperX) is strongly favored given the constraints:

- **Cost**: $0 vs $17-58 for APIs. The $20 budget is tight for API approaches and leaves no room for re-runs or quality iteration.
- **Accuracy**: For clean 2-speaker podcast audio, WhisperX with large-v3 + pyannote 3.1 matches or approaches API quality. The APIs' advantages are strongest on noisy/phone audio, which is not the case here.
- **Speaker labeling**: Pyannote voiceprints provide a free, local path to labeling Joe Hudson by name. This is a differentiator vs APIs (which mostly return "Speaker 1/2" unless using Speechmatics).
- **Iteration**: Local processing allows unlimited re-runs to tune parameters. API approaches charge per run.

Recommended experiments:
1. Run WhisperX on 2-3 downloaded episodes, evaluate transcript quality and diarization accuracy.
2. Test pyannote voiceprint matching to label Joe Hudson, using a known solo segment as the enrollment sample.
3. If WhisperX quality is insufficient, run the same episodes through AssemblyAI as a comparison baseline before committing budget.
