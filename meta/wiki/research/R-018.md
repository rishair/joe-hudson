# What link conventions exist in the current coach directory and what normalization is needed?

## Question

The `coach/` compendium uses inconsistent link conventions (per the user transcript: "the links in the coach directory are kind of haphazard"). The web app needs all internal links to resolve. R-018 catalogs what's actually in `coach/` and specifies the normalization the ingestion script must perform: which link formats appear (Obsidian-style `[[wikilinks]]`, markdown `[text](path)`, frontmatter `related:` arrays, bare path mentions, etc.), how they currently break (case sensitivity, missing extensions, wrong category prefixes, dangling links), and what canonical format the web app expects.

R-014 already noted that `_backlinks.json` is the inverse graph and that 100% of concerns lack frontmatter `related:` arrays despite linking heavily in body wikilinks. R-018 builds on this — characterizes the actual link surface so E-039 (the ingestion script) can normalize it correctly.

## Why This Matters

E-039 (the ingestion script) directly depends on this. The wrong normalization rules either produce broken links in the rendered wiki (silent failure) or rewrite valid content into invalid content (loud failure). R-018 specifies the rules before E-039 implements them, so E-039 is mechanical translation rather than discovery.

## Constraints and Scope

In scope:
- Catalog every distinct link format that appears in `coach/` (sample at least 20 files across categories)
- Specifically handle Obsidian-style alias syntax `[[slug|display text]]` — the display-text variant is common in handwritten notes and the renderer must show "display text" while linking to "slug"
- Count broken links (link target doesn't resolve to any file under `coach/`)
- Categorize broken links: case mismatch, missing file, wrong category, ambiguous (target exists in multiple categories)
- Specify the canonical web-app format (recommend: `[[slug]]` or `[[slug|display]]` with slug = `category/filename-without-extension`, resolved at render time)
- Specify the normalization rules E-039 must apply (mapping table from observed formats to canonical)

Out of scope:
- Fixing the source `coach/` directory itself (G-007's responsibility; R-014 already filed recommendations)
- Building the renderer that resolves the canonical format (E-042's job)
