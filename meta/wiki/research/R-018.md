# What link conventions exist in the current coach directory and what normalization is needed?

## Question

Catalog every distinct internal-link format that appears in `coach/`, count breakage, and specify the canonical web-app format plus the normalization rules E-039's ingestion script must apply.

## Sources

- Full scan of `coach/` (2,376 markdown files across 13 categories, excluding `_*.md`, `bin/`, `checkpoints/`)
- Audit script at `/tmp/audit-links.py` (saved JSON output at `/tmp/link-audit.json`)
- `coach/bin/rebuild-index.sh` — the existing index/backlinks regeneration script
- `coach/_index.md` (2,424 lines) and `coach/_backlinks.json` (28,921 lines) — derived files already produced by the rebuild script
- R-014 (the prior audit of `related:` coverage from 2026-05-22)
- Hand-sampled 26 files (2 per category) for body link inspection

No LLM API calls were made; this is a static analysis.

### Canonical web-app format (the renderer's input)

The wiki renderer (E-042's `WikiRenderer` component) consumes wikilinks in a single canonical form:

```
[[slug]]                  → render as <a href="/wiki/{category}/{slug}">{title-of-target}</a>
[[slug|display text]]     → render as <a href="/wiki/{category}/{slug}">{display text}</a>
```

The renderer looks up `category` from `slug` at render time using a `Map<slug, {category, title}>` built once from the ingest manifest. Because slugs are globally unique, the lookup is deterministic.

URLs use a category prefix in the path (`/wiki/concepts/limiting-belief`, not `/wiki/limiting-belief`) for two reasons: (a) the existing UX language in [[G-010]] explicitly shows category-prefixed URLs; (b) category prefixes carry useful information for the human reader and for navigation breadcrumbs. The slug-only link target in source markdown is purely an authoring convenience — the rendered link is fully qualified.

For broken links (target slug does not resolve to any file), the renderer renders them as a visibly-broken span — e.g., `<span class="wikilink-broken" title="no target: the-pucker-as-the-truth-marker">the-pucker-as-the-truth-marker</span>`. This is the loud-failure mode the goal requires: silent fallthrough hides corpus issues; visible broken links surface them for G-007.

### Normalization rules E-039 must apply

The ingestion script applies these transformations during the copy from `coach/` to `web-app/content/wiki/`. Each rule is deterministic and reversible (the source is never modified):

**Rule 1 — Body wikilinks: parse, validate, canonicalize.**

For every `[[...]]` match in the body of a copied file:

1. Split on `|` into `target` + optional `display`. Trim whitespace from each.
2. Split `target` on `#` into `slug-part` + optional `anchor`. (Anchor is preserved on the rendered link as `#anchor` but does not participate in resolution beyond the slug.)
3. Split `slug-part` on `/` into `category-part` + `slug`. If only one part, treat it as `slug` with no category constraint.
4. **Case-fold resolution:** if exact `slug` is not in the slug-to-file map, try `slug.toLowerCase()`. If found, use the canonical-case version. If not, mark link as broken.
5. Replace bare-space slugs (`[[5-Star Meetings]]`, `[[false dichotomy]]`) by trying both the as-is form AND the kebab-cased form (`5-star-meetings`, `false-dichotomy`). If kebab-case resolves, use it. (None of the 5 cases in the corpus currently resolve; all become broken-link spans. This rule is for forward-compat.)
6. Write back as canonical form: `[[slug]]` or `[[slug|display]]` or `[[slug#anchor]]` or `[[slug#anchor|display]]`. **Never write `category/slug` in the body** — categories are resolved at render time.
7. If the link is broken, leave it as-is in the markdown body but record it in the ingest manifest's `broken_links` list.

**Rule 2 — Frontmatter `related:` arrays: validate, canonicalize, mirror.**

For every entry in a `related: [...]` array:

1. Trim whitespace.
2. Case-fold resolve (same as Rule 1.4).
3. If the entry has a `/` prefix (`category/slug`), strip it — `related:` is slug-only by convention.
4. If the entry has spaces, attempt kebab-case resolution.
5. Replace with canonical lowercase slug if resolved; leave as-is if not.
6. Record any unresolved entries in the ingest manifest's `broken_related` list.

**Rule 3 — Markdown-syntax internal links (`[text](path.md)`): rewrite to wikilink.**

Only one instance in the corpus. The script detects relative `.md` paths in `[text](path)` and rewrites them to `[[slug|text]]` where `slug` is the path's basename without extension. (If text == slug, drop the pipe.)

**Rule 4 — External and image links: pass through unchanged.**

URLs starting with `http://`, `https://`, `mailto:`, `data:` and image embeds (`![alt](url)`) are not normalized.

**Rule 5 — Frontmatter preservation.**

Frontmatter is parsed (`key: value`, inline `[...]` arrays), `related:` is canonicalized per Rule 2, all other keys pass through verbatim. The script does NOT alter `id`, `title`, `aliases`, `tags`, `sources`, or any other frontmatter field.

**Rule 6 — Derived files (`_index.md`, `_backlinks.json`).**

The script regenerates these AFTER the copy from the normalized output. They are NOT copied from `coach/` directly because the normalized output's slugs may differ slightly (case fixes, kebab-case fixes) from the source. Two implementation options:

- Invoke `coach/bin/rebuild-index.sh` parameterized against `web-app/content/wiki/` (works because the script is ~50 lines of Python with hard-coded paths; needs a small refactor).
- Reimplement the regeneration in TypeScript inline (same algorithm; ~50 lines).

Either is fine; the requirement is that the derived files in the web-app match the normalized content, not the source.

### Specifications the renderer (E-042) needs

The renderer must:

1. Accept the canonical form `[[slug]]` or `[[slug|display]]` (with optional `#anchor`).
2. Resolve slug → `{category, title}` from an in-memory map built from the ingest manifest.
3. Render resolved links as `<a href="/wiki/{category}/{slug}{#anchor?}">{display ?? title}</a>`.
4. Render broken links visibly differently (`<span class="wikilink-broken">{display ?? slug}</span>`).
5. Per R-017's contract, accept a `linkBehavior` prop. With `linkBehavior="route"` (default), clicking navigates. With `linkBehavior="in-modal"`, clicking calls an `onLinkClick(slug)` callback instead of navigating.

The ingestion script produces, alongside the mirrored content, a manifest file `web-app/content/wiki/.ingest-manifest.json`:

```json
{
  "version": 1,
  "generated_at": "ISO timestamp",
  "files": {
    "concepts/limiting-belief": {
      "source_hash": "sha256...",
      "title": "Limiting Belief",
      "aliases": ["..."]
    }
  },
  "broken_links": [
    {"source": "concepts/foo", "target": "missing-bar", "reason": "no-target"}
  ],
  "broken_related": [...],
  "slug_to_category": {
    "limiting-belief": "concepts"
  }
}
```

The renderer loads `slug_to_category` into a `Map` at app startup (or per-route data fetch). Lookups are O(1).

## Implications

### For E-039 (the ingestion script)

E-039's blast radius and method are largely correct as currently written. The corrections this research forces:

1. **Corpus count is now 2,376 markdown files in 13 categories** (up from the audit's 1,860 estimate; the absorption pass between 2026-05-23 and 2026-05-24 added several hundred files). Update E-039's count.
2. **Integrity threshold should be raised from 90% to 97%**. The actual resolve rate is 98.6% (body) and 98.5% (related). The 90% target was a conservative guess; the corpus is healthier than R-014 predicted. Use 97% to leave a small buffer for ongoing absorption.
3. **The 6 normalization rules above are the spec.** The author of E-039 implements them as written; there is no ambiguity to resolve.
4. **No slug-collision handling is needed today.** Zero collisions exist in the corpus. If a future collision arises (e.g., G-007 absorbs a file with the same name into two categories), the ingest must fail loudly rather than silently picking one — add a "fatal on duplicate slug" check as forward-compat.
5. **Case-folding is the largest auto-fix.** Auto-fixing 22 case-mismatch links and 1 case-mismatch `related:` entry takes the broken count from 491 to 469 (body) and 232 to 231 (related). All other broken links are missing-target issues that belong on G-007's docket.
6. **The `_index.md` and `_backlinks.json` derived files should be regenerated from the normalized output, not copied verbatim.** See Rule 6.

### For E-042 (the renderer)

1. **Single canonical input form** simplifies the parser. Renderer accepts `[[slug]]`, `[[slug|display]]`, `[[slug#anchor]]`, `[[slug#anchor|display]]`. No category-prefixed form needs to be supported in the input.
2. **Slug-to-category lookup must be available at render time.** This is the `Map<string, {category, title}>` from the ingest manifest. The wiki repository (per [[coding-architecture]]) owns this map.
3. **Broken-link rendering is a UX decision** but the renderer must distinguish broken from resolved. Per R-017's `linkBehavior` contract, broken links in the modal context should also NOT trigger navigation (a broken link can't go anywhere).

### For G-007 (compendium ownership)

Recommendations docket additions (independent of R-014's existing docket):

1. **Auto-fix the 22 case-mismatch wikilinks in `coach/` source files** to match the canonical lowercase. This is mechanical and safe: the targets exist; the source just has wrong case. Files identified in the audit JSON. Predicted effort: 5 minutes via script.
2. **Auto-fix the 1 case-mismatch `related:` entry.** Trivial.
3. **The 469 missing-target body wikilinks are real corpus gaps.** Some are typos, some are references to articles that should exist but don't. Generate a CSV `target → reference count → source files` and triage. The most-referenced missing targets are the highest-leverage to author:
   - `the-pucker-as-the-truth-marker` (5 refs, missing)
   - `dont-use-view-as-technique` (4 refs)
   - `fix-the-team-by-fixing-the-leader` (4 refs)
   - `loving-the-emotion` (3 refs)
4. **Common-word link disambiguation**: links like `[[reads]]`, `[[essence]]`, `[[gaslighting]]`, `[[solitude]]`, `[[should]]` are too generic to be the slug of a single article. The authoring convention going forward should be specific-shape slugs (`[[essence-as-undefended-self]]` instead of `[[essence]]`) or piping with display text. The ~17 distinct generic words in the broken list should be triaged for whether to author the canonical article or rewrite the source link.

The link-normalization unknown is resolved. E-039 can be claimed and implemented as a mechanical translation of the 6 rules above. The renderer (E-042) has a single canonical input form to parse. No further research on link conventions is needed before either experiment runs.
