# Implement multi-turn test harness

## Hypothesis

A Python script can orchestrate the full eval loop: (1) load a client profile, (2) generate the client system prompt, (3) run a multi-turn conversation between client LLM and coach-under-test, (4) feed the conversation to a judge LLM with rubrics, (5) output structured JSON scorecards. This is the integration point where all prior experiments come together.

## Method

1. Create `eval/run.py` with the following architecture:
   - `load_profiles(path)` — reads all YAML profiles from `eval/profiles/`
   - `load_rubrics(path)` — reads all rubric YAMLs from `eval/rubrics/`
   - `load_anti_pattern_tests(path)` — reads anti-pattern test cases
   - `load_fidelity_checklist(path)` — reads the methodology checklist
   - `load_safety_criteria(path)` — reads safety hard-fail criteria from E-027
   - `run_conversation(coach_config, client_profile, max_turns=12)` — runs the multi-turn loop (using validated client prompts from E-028)
   - `score_conversation(conversation, rubrics, checklist, anti_patterns, safety_criteria)` — runs the judge with per-turn annotations
   - `output_scorecard(scores)` — writes JSON to `eval/results/`
2. Coach config format: a YAML or JSON file specifying system prompt, model, and retrieval config (if any)
3. Use Claude API via Anthropic SDK for both client LLM and judge LLM. Judge model selection per R-010 findings.
4. Judge uses the prompt structure from R-010, including per-turn annotations (not just per-conversation scores)
5. Output format: one JSON file per conversation (with per-turn annotations) + one aggregate summary
6. Safety criteria are evaluated as an override layer: any safety hard-fail produces a FAIL regardless of other scores
7. Estimate and document runtime cost: number of API calls, estimated tokens, estimated wall-clock time for full suite

Prototype on 2 profiles first. Verify the pipeline end-to-end. Then run all profiles.

## Success Criteria

- `eval/run.py` executes without error
- Running against 2 profiles produces 2 JSON scorecards with scores for all dimensions
- Scores are in the expected 1-3 range with evidence citations
- Aggregate summary shows per-dimension averages
- Total runtime is under 10 minutes for 2 profiles
