# Build resource-attribution modal split layout

## Hypothesis

Implementing the resource-attribution UX per R-017's chosen pattern produces the interaction the user specified: subtle in-flow indicator on each assistant message, click → split modal (left = wiki entry, right = resource list), inter-resource navigation in the modal, back-to-chat, expand-to-full-page. The wiki renderer from E-042 powers the left-panel; the per-message metadata from E-043 populates the right-panel.

## Method

1. Read R-017's UX pattern recommendation. Follow the chosen routing strategy (modal-as-route via parallel/intercepting routes, or modal-as-state with URL params for shareability).
2. Per-message indicator:
   - Add a small footer to each assistant message showing the resource count (e.g., `📎 5`)
   - Subtle styling: gray, small, does not break visual flow
3. Modal:
   - Triggered by clicking the indicator
   - Layout: 60% left (wiki entry), 40% right (resource list with previews)
   - Right panel: list of `{ title, category, snippet }` for each retrieved resource in that message
   - Click a right-panel item → updates the left panel to that resource (no full page reload)
   - Wikilinks within the left panel → navigate the left panel only (modal stays open, right panel unchanged)
4. Action affordances:
   - "Back to chat" (X or back button) → close the modal, return to the conversation at the originating message
   - "Expand" on the left-panel header → push a route to `/wiki/<slug>` (the E-042 full page), with a "back to chat at message N" affordance in that view
5. Reactive UI state (Zustand layer, not an architectural Store per the revised [[coding-architecture]] playbook — the modal has no persistent data behind it; Zustand here is purely UI state):
   - `useResourceModalState`: `{ openMessageId: string | null, focusedSlug: string | null, open(messageId, initialSlug), focus(slug), close() }`
6. Test: open a conversation with retrieval, click resource indicator, navigate between resources, click a wikilink in the left panel, hit expand, hit back to chat — verify each affordance works and state is preserved

## Success Criteria

- Resource indicator renders subtly (passes a "does not disrupt flow" eyeball test)
- Modal opens on click with the correct focused resource
- Switching between right-panel resources updates left panel without route change or flash
- Wikilinks inside the modal navigate the left panel only
- "Expand" pushes to `/wiki/<slug>` with a working "back to chat" affordance
- Closing the modal returns to the conversation scroll position at the originating message
- Modal is keyboard-accessible (escape closes, tab navigates)
