# Build resource-attribution modal split layout

## Hypothesis

Implementing the resource-attribution UX per R-017's chosen pattern produces the interaction the user specified: subtle in-flow indicator on each assistant message, click → split modal (left = wiki entry, right = resource list), inter-resource navigation in the modal, back-to-chat, expand-to-full-page. R-017 explicitly resolved this with: native `<dialog>` modal (NOT Next.js parallel/intercepting routes), `nuqs` URL-state, and resources read directly from R-016's `data-resources` part on the assistant message. The E-042 wiki renderer powers the left-panel via a new `linkBehavior="in-modal"` prop (E-042 adds the prop); the `data-resources` part on each `UIMessage.parts` array populates the right-panel.

## Method

1. **Add dependency:** `bun add nuqs` and wire `<NuqsAdapter>` at the root layout (see nuqs docs — for App Router this goes in `app/layout.tsx`).
2. **Per-message indicator** (`app/components/message.tsx` or wherever assistant messages render):
   - For each assistant message, find `message.parts.find(p => p.type === 'data-resources')` (typed via R-016's `CoachUIMessage`).
   - If present and resources non-empty, render a subtle bottom-of-message strip: small gray text "N sources" with a chevron icon. Click → calls `useResourceModal().open(message.id, resources[0].slug)`.
   - If no resources part (e.g., a pre-retrieval intro message), render nothing — do NOT render an empty strip.
3. **State hook** at `app/lib/state/use-resource-modal.ts`:
   ```typescript
   'use client';
   import { useQueryState } from 'nuqs';
   import { useChat } from '@ai-sdk/react';  // or read from wherever messages live
   import type { CoachUIMessage } from '@/app/lib/coach/types';

   export function useResourceModal() {
     const [messageId, setMessageId] = useQueryState('resourceModal', { history: 'push' });
     const [focusedSlug, setFocusedSlug] = useQueryState('focus', { history: 'push' });
     // Read messages from the chat layer (concrete source TBD per E-041's hook surface)
     const messages = useChatMessages() as CoachUIMessage[];
     const message = messages.find((m) => m.id === messageId);
     const resourcesPart = message?.parts.find((p) => p.type === 'data-resources');
     const resources = resourcesPart?.data?.resources ?? [];

     return {
       isOpen: !!messageId,
       messageId, focusedSlug, resources,
       open: (msgId: string, slug: string) => { setMessageId(msgId); setFocusedSlug(slug); },
       focus: (slug: string) => setFocusedSlug(slug),
       close: () => { setMessageId(null); setFocusedSlug(null); },
     };
   }
   ```
4. **Modal component** at `app/components/resource-modal.tsx`:
   - `'use client'`. Reads `useResourceModal()`. Returns `null` when not open.
   - Native `<dialog ref>` with `useEffect(() => { ref.current?.showModal(); }, [])`. Renders via `createPortal` into a `#modal-root` div (added once to root layout).
   - Layout: 65% left (focused wiki entry), 35% right (resource list).
   - Left panel: renders the focused-slug page via E-042's `<WikiRenderer slug={focusedSlug} linkBehavior="in-modal" onLinkClick={(slug) => focus(slug)} />`. Header shows page title + "Expand" button + close button (X).
   - Right panel: list of resource cards (title, category badge, 1–2 line snippet) — `<button>` per item, `aria-current={slug === focusedSlug}`, click → `focus(slug)`. Focused card has a left border + tinted background.
   - `aria-labelledby` on `<dialog>` points to the left-panel title.
   - `aria-live="polite"` region: "Now viewing: <title>" updates on focus changes.
   - Modal sizing: ~90vw × 85vh, max-width 1400px.
5. **Expand action** (left-panel header "Expand" button):
   ```typescript
   function expand() {
     const slug = focusedSlug;
     const msgId = messageId;
     close();  // clears nuqs state first
     router.push(`/wiki/${slug}?fromChat=${msgId}`);
   }
   ```
6. **Render the modal** at the root of the chat surface (e.g., `app/page.tsx`):
   ```tsx
   export default function Page() {
     return (
       <>
         <ChatSurface />
         <ResourceModal />  {/* renders null when no URL state */}
       </>
     );
   }
   ```
   Add `<div id="modal-root" />` to `app/layout.tsx` for the portal target.
7. **Test plan** (vibe + automated):
   - Open conversation with retrieval. Confirm bottom strip shows "N sources" on each assistant message after the retrieval phase.
   - Click strip → modal opens with first resource focused, URL updates to `?resourceModal=<id>&focus=<slug>`.
   - Click a right-panel resource → left panel changes, URL `focus` updates, modal stays open, no flash.
   - Click a wikilink inside the left panel (an internal `[[...]]` link from the focused entry) → left panel changes to the linked entry, modal stays open.
   - Click Expand → URL becomes `/wiki/<slug>?fromChat=<msgId>`, full-page wiki view renders, "Back to chat at message N" link appears.
   - Click Back-to-chat → returns to chat, scroll position restored at message N (chat surface reads `?scrollToMessage=` on mount).
   - Browser back from open modal → modal closes (nuqs `history: 'push'` recorded each state change).
   - Escape key in modal → close.
   - Tab navigation stays inside modal (focus-trap via native `<dialog>`).
   - Share URL with `?resourceModal=...&focus=...` to another tab/browser → modal opens to the correct state on load.

## Success Criteria

- Resource strip renders subtly per assistant message that has a `data-resources` part; passes a "does not disrupt flow" eyeball test.
- Modal opens on click with the correct focused resource and the full resource list visible in the right panel.
- Switching between right-panel resources updates the left panel without route change or visible flash; URL `?focus=<slug>` updates.
- Wikilinks inside the modal's left panel navigate the left panel only (right panel and modal state unchanged).
- "Expand" pushes to `/wiki/<slug>?fromChat=<msgId>` and that view renders a working "Back to chat at message N" link.
- Closing the modal (close button, escape, browser back, or backdrop click) returns to the conversation with scroll position at the originating message.
- Modal is keyboard-accessible: Escape closes, Tab/Shift+Tab cycle within the dialog, focus returns to the resource strip that opened the modal.
- Direct URL load (`/?resourceModal=<msgId>&focus=<slug>`) opens the modal to the correct state on first paint.
