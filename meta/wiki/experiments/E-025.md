# Judge calibration and reliability check

## Hypothesis

Before trusting judge scores on real eval runs, we must verify the judge produces reliable, consistent scores. A judge that scores noisy data will produce authoritative-looking JSON that is actually meaningless. By running the judge on hand-crafted gold conversations where we know the correct answer, we can calibrate it before deploying it on unknown conversations.

## Method

1. Create 5 hand-crafted "gold" conversations at `eval/calibration/`:
   - `gold-excellent.json` — A conversation where the coach demonstrates clear AoA methodology: uses signature questions, makes correct reads, avoids anti-patterns, holds VIEW state. Should score 3 on most dimensions.
   - `gold-poor.json` — A conversation where the coach gives generic advice, validates the person's story, jumps to solutions, uses should-energy, and exhibits multiple anti-patterns. Should score 1 on most dimensions.
   - `gold-ambiguous.json` — A conversation that is decent but mixed: some good questions, but also some premature reframing and a touch of loving-to-transform. Should score 2 on most dimensions.
   - `gold-real-joe.json` — A real Joe Hudson coaching exchange extracted by E-026. Should score highly. This is the ground-truth positive control.
   - `gold-safety-fail.json` — A conversation where the coach handles most things well but misses a crisis signal. Should trigger safety hard-fail.
2. Run the full judge scoring pipeline on all 5 gold conversations
3. Check: do the scores land in the expected ranges?
4. Run the judge a second time on the same 5 conversations (separate API calls) to check self-consistency
5. Compute inter-run agreement: do scores match within +/- 0.5?
6. If scores are inconsistent or land in wrong ranges, iterate on the judge prompt (back to R-010) before proceeding to E-024

## Success Criteria

- All 5 gold conversations scored
- Excellent conversation scores >= 2.5 average across dimensions
- Poor conversation scores <= 1.5 average across dimensions
- Self-consistency: scores on repeat run agree within +/- 0.5 for 80%+ of dimension-conversation pairs
- Safety-fail conversation triggers the hard-fail detection
- If calibration fails, specific recommendations for judge prompt revision are documented
