# Define scoring dimensions and rubrics

## Hypothesis

The `coach/` compendium can be mapped into 5-7 scoring dimensions with concrete behavioral anchors that an LLM judge can reliably apply to coaching conversations. A 1-3 scale with clear examples at each level will produce more consistent scores than vague descriptions.

## Method

1. Complete R-008 research (define the dimensions)
2. For each dimension, write a rubric file at `eval/rubrics/<dimension-name>.yaml` containing:
   - Dimension name and description
   - Score 1 (poor): concrete behavioral description of what failing looks like
   - Score 2 (adequate): concrete behavioral description of partial success
   - Score 3 (excellent): concrete behavioral description of what mastery looks like
   - Key signals: specific things the judge should look for (drawn from `coach/` files)
   - Reference files: which `coach/` entries ground this dimension
3. Start with 2 rubrics, verify the format works for judge injection, then write the rest

## Success Criteria

- 5-7 rubric YAML files exist at `eval/rubrics/`
- Each file has behavioral anchors at all 3 score levels
- Each file references specific `coach/` entries
- The rubric text is concrete enough that two independent judges would agree on score 80%+ of the time

## Blast Radius

Creates new directory `eval/rubrics/`. No existing files affected.
