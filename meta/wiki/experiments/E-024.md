# Run baseline eval against naive system prompt

## Hypothesis

A minimal system prompt ("You are a coach inspired by Joe Hudson's Art of Accomplishment") will produce measurably lower scores than the target we are building toward. This establishes the floor. If the eval cannot differentiate a naive prompt from a well-crafted one, the eval is broken. The baseline run validates the eval's discriminative power.

## Method

1. Create `eval/baselines/naive.yaml`:
   ```yaml
   name: naive-baseline
   model: claude-sonnet-4-6
   system_prompt: |
     You are an AI coach inspired by Joe Hudson's Art of Accomplishment methodology.
     Help the user explore their emotions, wants, and limiting beliefs.
     Ask open-ended questions. Be empathetic and curious.
   retrieval: none
   ```
2. Run `eval/run.py` with this config against all 15 client profiles
3. Record aggregate scores per dimension
4. Analyze: which dimensions score lowest? Which anti-patterns does the naive prompt fall into?
5. Document specific failure examples (conversations where the naive coach fails obviously)

## Success Criteria

- Full eval run completes against all 15 profiles
- Aggregate scores are interpretable and vary across dimensions (not all 1s or all 3s)
- At least 3 dimensions show clear room for improvement (average < 2.0)
- At least 5/10 anti-pattern tests trigger failures
- Results suggest specific directions for the coach system prompt (what to add to improve specific dimensions)

## Blast Radius

Creates `eval/baselines/naive.yaml` and writes results to `eval/results/baseline-naive/`. No existing files affected.
