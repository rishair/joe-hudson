# Podcast episode download via iTunes API URLs

## Hypothesis

The Simplecast CDN audio URLs from the iTunes Lookup API (obtained in [[E-002]]) are direct, stable links that work for all 167 episodes, including the oldest from 2020. Downloading via curl with these URLs will produce valid MP3 files suitable for transcription.

## Method

The original plan was to use the PodcastIndex.org API. This was abandoned in favor of the iTunes Lookup API approach, since [[E-002]] already obtained complete data for all 167 episodes with direct audio URLs, requiring no API key or registration.

1. Read `meta/raw/itunes-episode-list.tsv` (167 episodes with date, title, duration, and Simplecast CDN URL).
2. Test single download of the oldest episode (2020-10-26): `curl -sL -o <path> <url>`.
3. Test batch of 3 episodes with directory creation and filename sanitization.
4. Verify downloaded files with `file` command.

## Result

**3 of 3 test episodes downloaded successfully** via curl from Simplecast CDN URLs.

| Episode | Date | Size | Bitrate | Format |
|---------|------|------|---------|--------|
| Introduction to VIEW | 2020-10-26 | 45 MB | 128 kbps | MPEG Layer III, 44.1 kHz, JntStereo |
| Wonder | 2020-11-04 | 48 MB | 128 kbps | MPEG Layer III |
| Impartiality | 2020-11-05 | 47 MB | 128 kbps | MPEG Layer III |

All URLs returned HTTP 200 with no authentication required. The oldest episode (from October 2020) downloaded without issue. The Simplecast CDN URLs contain tracking parameters (`aid=rss_feed`, `awCollectionId`, `awEpisodeId`, `feed`) but do not expire.

The download command used: `curl -sL -o "$dir/$safetitle.mp3" "$url"` with title sanitization to remove `/` and `:` characters.

**Scaling estimate**: 167 episodes at ~47 MB average = ~7.8 GB total. At observed download speeds (~5-10 MB/s), a full download takes approximately 15-25 minutes.

## Analysis

The podcast download pipeline is validated. The iTunes API URLs from [[E-002]] work for the full episode history. No PodcastIndex registration or API key is needed.

The podcast audio is at 128 kbps (higher quality than the YouTube audio at 96 kbps from [[E-008]]). This is expected since podcasts are distributed at their original encoding quality, while YouTube re-encodes uploaded audio.

**Directory structure**: Episodes are saved as `transcripts/{YYYY-MM-DD}_{title}/{title}.mp3`. Title sanitization replaces `/` and `:` with `-` and truncates at 80 characters to avoid filesystem issues.

**Boundary conditions**: Tested with 3 episodes (oldest and second/third oldest). The URLs contain no obvious expiration token, but Simplecast could rotate CDN endpoints in the future. The complete download should be run soon to capture all episodes while URLs are confirmed working.

**PodcastIndex API (original plan)**: Not attempted. The iTunes API approach from [[E-002]] rendered it unnecessary. PodcastIndex remains a viable fallback if iTunes URLs ever stop working.

## Dead Ends

None. The original PodcastIndex approach was abandoned before testing because a simpler alternative (iTunes API URLs from [[E-002]]) was already available and proven.
