# Move web-app deploy to ris.hair account, rename to joebot, serve at joebot.ris.hair custom domain

## Hypothesis

The web app can be redeployed from the rishair.com account (worker `joe-hudson-coach`) to the ris.hair account (worker `joebot`) at the custom domain `joebot.ris.hair`, with no change to app behavior. Consolidating onto the ris.hair account puts the app and the G-012 PaaS on one account and gives the app a real domain instead of a `workers.dev` URL.

## Method

Depends on [[E-054]] (so the moved deploy includes the variant selector) and [[REQ-005]] (the `CLOUDFLARE_JOEBOT_WORKERS_TOKEN` with Workers Scripts + Account Settings + zone Workers Routes + zone DNS edit on ris.hair). Verified prerequisites as of 2026-05-25: token deploys Workers and has Workers Routes on ris.hair; **DNS Edit was still pending** (needed for the custom-domain record) — confirm it is granted before running.

1. **Config edits in `web-app/wrangler.jsonc`:**
   - `name`: `joe-hudson-coach` → `joebot`
   - `services[0].service` (WORKER_SELF_REFERENCE): → `joebot` (must match `name`)
   - `vars.APP_URL`: → `https://joebot.ris.hair`
   - Add a custom-domain route: `"routes": [{ "pattern": "joebot.ris.hair", "custom_domain": true }]`
2. **Account targeting.** The deploy must run against account `75e26858f8d36b8fad28ad311dd0f360` (ris.hair) using `CLOUDFLARE_JOEBOT_WORKERS_TOKEN`. Set `CLOUDFLARE_ACCOUNT_ID` + `CLOUDFLARE_API_TOKEN` in the deploy environment to the joebot account/token (or pass `--account-id`), NOT the rishair.com values still in `.env`.
3. **Secrets on the new account's worker.** `wrangler secret put OPENROUTER_API_KEY` and `ANTHROPIC_API_KEY` against the `joebot` worker (one-time). These do not transfer between accounts.
4. **Deploy:** `bun run deploy:cf` (runs ingest → prebuild:cf → build:cf → opennextjs-cloudflare deploy) with the joebot account/token in env.
5. **Verify:** `joebot.ris.hair` resolves with a valid cert (poll issuance, ~10-30s typical), returns 200, the welcome flow shows, a coach turn streams, the variant selector works. Re-run [[Q-001]]/[[Q-002]] against the new URL.
6. **CI workflow** (`.github/workflows/deploy.yml`): update the `CLOUDFLARE_ACCOUNT_ID` GitHub secret + `APP_URL` var to the joebot account/domain, OR note CI is deferred (local deploy covers it; GitHub Actions was a v2 nice-to-have per REQ-003).
7. **Retire the old deploy:** leave `joe-hudson-coach` on the rishair.com account running until `joebot.ris.hair` is confirmed, then optionally delete it. Update [[REQ-003]] to mark the old Workers token retired.

## Success Criteria

- `joebot.ris.hair` serves the app over HTTPS with a valid cert, 200 on `/`.
- Coach chat (with variant selector), streaming, modal, persistence all work on the new URL (no regression).
- The worker is named `joebot` on the ris.hair account; secrets set; `APP_URL` updated.
- The custom-domain route is bound; no `workers.dev` dependency for the canonical URL.

## Blast Radius

Modifies `web-app/wrangler.jsonc` (name, self-reference, vars, routes), possibly `.github/workflows/deploy.yml` (account/URL). Creates a NEW worker `joebot` on the ris.hair account and a DNS record + cert for `joebot.ris.hair` on the ris.hair zone (the only DNS write; scoped to that hostname). Does NOT change app code, coach logic, or `coach-app/`. The old `joe-hudson-coach` worker on the rishair.com account is left intact until cutover is confirmed. Must run AFTER [[E-054]] lands (depends_on) so it does not race the selector deploy; renaming creates a new worker, so the old one is not overwritten.
