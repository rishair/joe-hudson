# Implement warm-somatic redesign of the web app shell and fix mobile responsiveness

## Hypothesis

Applying the [[frontend-aesthetics]] playbook with a committed warm-somatic theme will transform the app from the default AI-slop baseline into a surface that reads warm, calm, and human, while fixing the [[Q-001]] mobile defects, with no loss of function. The aesthetic carries the coaching intent: a person should feel they can slow down and be honest, not that they are filling in a form.

## Method

Read the [[frontend-aesthetics]] playbook first and follow its recipe. Depends on [[E-054]] (the selector must already be wired; this experiment restyles it, it does not build the plumbing).

1. **State the design choices before coding** (playbook rule). Lock: Fraunces for display/headings; a distinctive non-default humanist sans for body (NOT Inter/Roboto/Arial/system — e.g. a warm grotesque or IBM Plex Sans, state the pick). Palette as CSS variables: warm sand/cream background, deep ink/espresso text, one terracotta-ochre accent. Type scale with large jumps and high weight contrast. Load fonts from Google Fonts (or self-host for the Cloudflare bundle; mind the bundle ceiling per [[cloudflare-deploy]]).

2. **Establish design tokens.** Define all color/space/type/radius as CSS variables in one place (`globals.css` or a tokens module). Everything downstream references tokens, so a later re-skin is a token swap (playbook iteration model). Optionally author a `<warm_somatic_theme>` constraint block saved in-repo (nice-to-have).

3. **Restyle the shell**, surface by surface, preserving behavior:
   - Chat view: generous whitespace, editorial rhythm, warm message bubbles, distinct but quiet user/coach treatment. Keep streaming ([[E-052]]) and the "N sources" affordance + modal ([[E-044]]) working.
   - Selector ([[E-054]]): place it in the new shell tastefully (header or a calm settings affordance), restyle to theme.
   - Welcome flow ([[E-045]]), wiki view ([[E-042]]), sidebar/history: restyle to theme.

4. **Fix mobile (the [[Q-001]] defects).** No horizontal overflow at 360-414px; the sidebar collapses or adapts (drawer/toggle) so the chat input is full-width and usable; selector reachable. Test at 412px and 375px.

5. **Motion budget.** One orchestrated page-load with a soft staggered fade (`animation-delay`); gentle message-in transition. Use Motion (React) if it is light on the bundle, else CSS-only. No flashy scattered micro-interactions.

6. **Background.** Subtle warm depth (soft gradient or paper texture), not a flat fill.

7. **Local verification** across desktop + mobile viewports: every prior surface works, no console errors, streaming + modal + persistence + selector intact.

8. **Deploy** per [[cloudflare-deploy]] / [[E-046]] (`bunx wrangler deploy` or the established command). Confirm the live URL serves the redesign and the bundle is under the Cloudflare ceiling (watch font/asset weight). Record the deployed URL and bundle size.

## Success Criteria

- The deployed app uses the warm-somatic theme: distinctive non-default fonts (Fraunces + chosen body face), warm-neutral background, deep ink text, single warm accent, all via CSS variables.
- No primary use of Inter/Roboto/Arial/system fonts; no purple-gradient-on-white; no flat default shell.
- No horizontal overflow at 412px and 375px; sidebar adapts; chat input usable on mobile (the [[Q-001]] defects are fixed).
- Chat/streaming, resource modal, persistence, welcome, wiki view, and the coach-variant selector all work post-redesign (no regression).
- Change is live at the Cloudflare URL, bundle under ceiling.
- [[Q-003]] confirms the above on desktop and mobile.

## Blast Radius

Touches the web-app presentation layer broadly: `web-app/app/globals.css` (or tokens module), `web-app/app/page.tsx`, `web-app/app/layout.tsx` (font loading), any component/CSS under `web-app/app/`, and `next.config.js` only if font/asset handling needs it (preserve the COOP/COEP headers from [[cloudflare-deploy]] — do not drop them). MUST NOT change coach behavior: leave `web-app/app/api/chat/route.ts` coach logic, `web-app/app/lib/coach/*` (profiles, prompts, retrieval, safety, cost-cap), and `coach-app/` untouched except for styling hooks. Redeploys the live Worker, replacing the current deploy. Sequenced after [[E-054]] (depends_on) to avoid `page.tsx` contention. Coordinate so this does not land mid-vibe-check (a redeploy is fine between sessions; the user is told when it ships).
