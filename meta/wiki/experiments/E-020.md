# Build 15 simulated client profiles from coach/concerns/

## Hypothesis

Each of the 21 `coach/concerns/` entries can be converted into a simulated client profile that drives a client LLM to produce realistic multi-turn conversations testing specific coaching capabilities. The profile schema from R-009 will generate conversations that exercise specific reads, moves, and anti-patterns.

## Method

1. Read all 21 `coach/concerns/` files
2. Select 15 that offer the most diverse coaching territory
3. For each, write a profile YAML at `eval/profiles/<concern-id>.yaml` containing:
   - id, concern (maps to coach/concerns/ filename)
   - scenario_type: one of happy_path, resistance, crisis, edge_case, red_herring
   - opening_statements: 2-3 variants of what the person might say (drawn from "What Someone Says" sections)
   - emotional_state: felt quality of how they show up
   - resistance_patterns: how they deflect, intellectualize, or avoid (drawn from the concern's coaching path)
   - traps: what the coach should NOT do (drawn from "What Joe Does Not Do" sections and related anti-patterns)
   - expected_territory: concerns, reads, moves, concepts, frameworks, anti_patterns_to_avoid
4. Ensure distribution across scenario types: 3 happy_path, 4 resistance, 3 crisis, 3 edge_case, 2 red_herring. (Audit note: happy path is the easiest case and least likely to surface problems. Resistance, crisis, and edge cases are where the eval earns its keep.)
5. Write a client system prompt template at `eval/client-prompt-template.md` that takes a profile and produces the client LLM instructions

## Success Criteria

- 15 profile YAML files exist at `eval/profiles/`
- All 5 scenario types are covered
- Each profile references real `coach/` file IDs in expected_territory
- Client prompt template exists and produces consistent client behavior when tested manually

## Blast Radius

Creates new directory `eval/profiles/` and file `eval/client-prompt-template.md`. No existing files affected.
