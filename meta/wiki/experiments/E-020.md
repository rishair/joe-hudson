# Build 15 simulated client profiles from coach/concerns/

## Hypothesis

Each of the 21 `coach/concerns/` entries can be converted into a simulated client profile that drives a client LLM to produce realistic multi-turn conversations testing specific coaching capabilities. The profile schema from R-009 will generate conversations that exercise specific reads, moves, and anti-patterns.

## Method

1. Read all 21 `coach/concerns/` files
2. Select 15 that offer the most diverse coaching territory
3. For each, write a profile YAML at `eval/profiles/<concern-id>.yaml` containing:
   - id, concern (maps to coach/concerns/ filename)
   - scenario_type: one of happy_path, resistance, crisis, edge_case, red_herring
   - opening_statements: 2-3 variants of what the person might say (drawn from "What Someone Says" sections)
   - emotional_state: felt quality of how they show up
   - resistance_patterns: how they deflect, intellectualize, or avoid (drawn from the concern's coaching path)
   - traps: what the coach should NOT do (drawn from "What Joe Does Not Do" sections and related anti-patterns)
   - expected_territory: concerns, reads, moves, concepts, frameworks, anti_patterns_to_avoid
4. Ensure distribution across scenario types: 3 happy_path, 4 resistance, 3 crisis, 3 edge_case, 2 red_herring. (Audit note: happy path is the easiest case and least likely to surface problems. Resistance, crisis, and edge cases are where the eval earns its keep.)
5. Write a client system prompt template at `eval/client-prompt-template.md` that takes a profile and produces the client LLM instructions

## Success Criteria

- 15 profile YAML files exist at `eval/profiles/`
- All 5 scenario types are covered
- Each profile references real `coach/` file IDs in expected_territory
- Client prompt template exists and produces consistent client behavior when tested manually

## Blast Radius

Creates new directory `eval/profiles/` and file `eval/client-prompt-template.md`. No existing files affected.

### Design decisions worth flagging

**1. Selection criterion was "distinctiveness of Joe-signature move," not "concern severity."** The eval needs profiles that test specific reproducible moves the coach must or must not make. `am-i-being-selfish` was chosen as a happy-path because Joe's refusal-to-debate is one of the cleanest signature behaviors in the entire compendium; a coach that engages the verdict question fails the test even if the conversation feels warm and productive.

**2. Happy paths are not "easy."** Per the eval-design playbook principle 7 (favor hard cases over easy ones), the 3 happy-path profiles still test specific failure modes: `happy-trust-myself-001` will trap a coach who engages the intellectual epistemology; `happy-am-i-selfish-001` will trap a coach who debates whether the want is selfish; `happy-need-boundaries-001` will trap a coach who gives mechanical scripts. The "happy" refers to the client's openness, not the coach's task.

**3. Red-herring scoring guidance is encoded as a YAML block, not a free-text description.** Each red-herring profile has a `red_herring_scoring` block with `partial_credit_if`, `full_credit_if`, `no_credit_if`, and `no_credit_also_if` clauses. This is an extension to R-009's red-herring schema that makes the scoring rule explicit so the judge does not have to reconstruct it. R-009 anticipated partial credit for landing on surface, full credit for landing on deeper; the YAML block also names two negative paths.

**4. `mid_conversation_resistance` is written as conditional narrative, not a flat list.** R-009 specified the field but did not specify the form. I wrote each as a 4-8 sentence "if the coach does X, the client does Y; if the coach holds the steadiness, then Z" narrative. This is testable by the client LLM (it has conditional pathways) and by the judge (it documents the gating). E-028 will validate whether this conditional structure produces consistent client behavior.

**5. The client prompt template strips judge-only fields explicitly.** A documented list of fields stripped (`golden_path`, `expected_territory`, `traps`, `safety_hard_fails`, `red_herring_scoring`, `seam_documentation`, `provenance`, `references`) is in the template's "Fields STRIPPED" section. This codifies the R-009 visibility split as enforceable harness behavior.

**6. The "edge case denial-of-the-concern" profile (`edge-not-anxious-001`) tests a session whose successful outcome looks small from outside.** The breakthrough is the client noticing once that "fine" has a quality. Per the concern file, fear differentiates out of numbness later, not in this session. The expected_scores for this profile will not all be 3; the per-dimension judge needs to know that a "small noticing" session is a success here, not a failure. This is a calibration test case for E-025.

**7. Trap descriptions name the failure mode AND the why_wrong.** Each trap has a description and a why_wrong rationale that cites the concern file or anti-pattern file by reference. This makes the trap inspectable -- a future maintainer can see why a behavior was marked a trap. It also feeds E-025 directly: the judge can be evaluated on whether it detected the trap and whether its evidence is consistent with the why_wrong.

## Analysis

### What the profiles cover that previous artifacts did not

The 15 profiles now operationalize the R-008 dimensions in testable form:
- D1 Perceptual Accuracy: every profile names specific reads the coach should activate, and which read would be wrong (e.g., reading anger as conflict when it is grief).
- D2 Intervention Quality: profiles encode the timing-sensitivity tests (e.g., `edge-not-anxious-001` traps a coach who pushes for fear before numbness has been felt).
- D3 Coaching Stance: the resistance patterns of `resistance-inner-critic-001` and `edge-dont-have-time-001` specifically test whether the coach can stay in VIEW under provocation.
- D4 Methodology Fidelity: `expected_territory.questions` lists signature questions the coach should deploy; `expected_territory.moves` lists the appropriate moves. The judge can compare what was used to what was expected.
- D5 Anti-Pattern Avoidance: every profile has a `traps` block with explicit anti-pattern refs, supplying ready-made anti-pattern test cases.
- D6 Effectiveness/Depth: every profile has a `breakthrough_condition` that is the testable hypothesis -- did anything actually shift, scored from client-turn evidence per R-008's design.

### Tests against the eval-design playbook

- Principle 2 (ground truth from the real thing): MET via R-009's design -- profiles are structured projections of `coach/concerns/` files, not invented archetypes.
- Principle 3 (test appropriateness, not presence): MET via the `mid_conversation_resistance` conditional narrative -- the breakthrough requires the move at the right time, not just the move.
- Principle 4 (safety is not quality): MET via the 3 crisis profiles being safety-relevant and gated on E-027's hard-fail criteria.
- Principle 6 (validate synthetic test subjects): NOT YET MET -- that is E-028.
- Principle 7 (favor hard cases over easy ones): MET via the 4 resistance + 3 crisis + 3 edge_case + 2 red_herring = 12 hard cases out of 15. Happy path is 3/15 (20%) -- below the audit-flagged risk of over-indexing.

### Risks / open questions for E-025 calibration

1. **Breakthrough condition tightness.** Each profile's `breakthrough_condition` is a multi-clause specification. If the client LLM softens on partial fulfillment, the eval will systematically over-reward coaches. If it requires perfect fulfillment, the eval will systematically under-reward. E-028 will measure this.

2. **Resistance fidelity across turns.** Each profile encodes 5-7 resistance patterns. A client LLM that drifts off the patterns mid-conversation will produce inconsistent eval results. E-028's character-consistency test addresses this.

3. **Happy-path "small noticing" sessions.** Profiles like `edge-not-anxious-001` end with a small recognition rather than a dramatic arc. The judge needs to score these as successes; if the judge has a bias toward dramatic-arc success, these will be under-scored. E-025 should include `edge-not-anxious-001` in calibration.

4. **Red-herring scoring sensitivity.** The two red herrings (`we-never-fight`, `if-i-had-more-money`) have 4-tier scoring (partial, full, no-credit, no-credit-also). If the judge collapses to binary "did the coach surface the deeper material yes/no" it will miss the gradient. E-025 should test this.

5. **Edge-case `methodology-questioning` traps.** `edge-dont-have-time-001` and `edge-parts-work-001` test whether the coach can refuse the framework-comparison or methodology-defense invitation. If the judge rewards "engaging respectfully with the question" without noticing the dichotomy was conceded, these will produce false positives. E-025 should test this.
