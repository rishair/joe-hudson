# Set up Cloudflare Tunnel + Access on Hetzner with a hello-world subdomain

## Hypothesis

Following R-019's recipe, a Cloudflare Tunnel can be installed on the Hetzner box as a `systemd` system service, routed to a single hello-world local port, and protected by Cloudflare Access. End state: hitting `hello.play.ris.hair` in an incognito browser shows the CF Access SSO page; signing in with the user's Google account passes through to a "hello world" page served from a local port.

This establishes the infrastructure foundation before building the CLI in E-048.

## Method

1. Read R-019's findings; follow its setup recipe.
2. Install `cloudflared` on the Hetzner box.
3. Create the Cloudflare Tunnel via dashboard or `cloudflared tunnel create`.
4. Write the initial `~/.cloudflared/config.yml` with a single ingress for `hello.play.ris.hair` → `http://localhost:8080`.
5. Install `cloudflared` as a `systemd` system service (`cloudflared service install`); verify it survives a reboot test (or simulate via `systemctl restart`).
6. Configure Cloudflare Access policy for `*.play.ris.hair` (per R-019 — wildcard vs per-subdomain).
7. Start a hello-world HTTP server on port 8080 (a 5-line Bun script).
8. Verify in incognito browser:
   - `hello.play.ris.hair` → CF Access SSO page
   - Sign in → hello world
   - Verify the Hetzner box has NO public ports exposed for the tunnel (only outbound connections)

## Success Criteria

- `systemctl status cloudflared` shows active+running, survives `systemctl restart cloudflared`
- `hello.play.ris.hair` resolves via Cloudflare and serves a CF Access page to unauthenticated requests
- User can sign in via Google/GitHub SSO and reach the hello-world page
- Anyone else hitting the URL stays at the Access page (verified by signing in with a different account — should be denied)
- `netstat -tlnp` on the Hetzner box shows no new public-bound ports added for the tunnel
- `~/.cloudflared/config.yml` is committed to a known location and documented

## Blast Radius

- Installs `cloudflared` system-wide on Hetzner (small dep, well-trusted)
- Adds a `systemd` system service
- Adds Cloudflare-side resources: a Tunnel, a DNS record for `*.play.ris.hair` (or `hello.play.ris.hair`), an Access policy
- Adds `~/.cloudflared/config.yml`
- Does NOT modify any existing project, port, or DNS record outside `play.ris.hair`
