# Build repeatable ingestion script copying coach to web app content dir with link normalization

## Hypothesis

A Bun/TypeScript script can copy the `coach/` directory into `web-app/content/wiki/`, normalize all internal links per R-018's rules, and be safely re-runnable (idempotent on unchanged files, reflecting changes on changed files, never duplicating or destroying user-side content). The output is consumed by both the chat's tool layer (E-043) and the wiki UI (E-042).

## Method

1. Read R-018's findings for the normalization spec.
2. Create `web-app/scripts/ingest-coach.ts`:
   - Reads from `coach/` (relative to repo root)
   - Writes to `web-app/content/wiki/` (creating if absent)
   - For each `.md` file: parse frontmatter, normalize body links per R-018, write to mirrored path
   - Compute a content hash per source file; skip writing if the output matches (idempotent)
   - Maintain `web-app/content/wiki/.ingest-manifest.json` mapping source → hash → output path
   - On re-run: detect removed source files and remove their outputs; detect new source files and add them; detect changed source files and re-process them
3. Add an integrity check at the end: re-resolve every internal link in the output and report any that don't point to an existing file.
4. Test ramp (corpus is **2,376 markdown files** in 13 standard categories as of 2026-05-24, per R-018's verified count — up from R-014's 1,059 mid-month and the audit's ~1,860 — G-007 absorption has continued):
   - First run on 3 files (a category subset). Verify normalization is correct per R-018's Rules 1–6.
   - Then 30 files (a full category). Verify integrity check is clean.
   - Then full `coach/` (~2,376 files). Verify total runtime is reasonable (target: under 60s for the full corpus on local hardware) and report broken-link count.
   - Coordinate with `coach/bin/rebuild-index.sh` — the ingestion script regenerates `_index.md` and `_backlinks.json` in the target directory AFTER copy (per R-018 Rule 6), since case-fixes and kebab-fixes during normalization mean the source's derived files can be inconsistent with the normalized output. Either parameterize `rebuild-index.sh` against the output dir or reimplement the ~50-line algorithm inline in TypeScript.
5. Add an npm/bun script alias: `bun run ingest` from `web-app/`.

## Success Criteria

- `bun run ingest` from `web-app/` produces `web-app/content/wiki/` with the expected file count (matches `coach/` markdown count; **2,376 as of 2026-05-24 per R-018**)
- Re-running with no `coach/` changes produces zero file writes (manifest skip-logic works)
- Modifying one `coach/` file, then re-running, produces exactly one file write plus updated `_index.md` and `_backlinks.json`
- Integrity check at end: **at least 97% of internal links resolve** (R-018 measured actual rate at 98.6% body / 98.5% related; 97% leaves a small buffer for ongoing absorption noise)
- After applying R-018's case-fold normalization (Rule 1.4), at least 22 case-mismatch body wikilinks and 1 case-mismatch `related:` entry are auto-resolved in the output
- `web-app/content/wiki/.ingest-manifest.json` is the only "scratch" file outside the mirrored content; its shape matches R-018's spec (includes `slug_to_category` map, `broken_links` list, `broken_related` list)
- The script fails loudly (non-zero exit) if it encounters a duplicate slug across categories — R-018 verified 0 collisions today, but this guards against future absorption introducing one
- `_index.md` and `_backlinks.json` are regenerated from the normalized output (NOT copied from `coach/`); per R-018 Rule 6

## Blast Radius

Creates `web-app/scripts/ingest-coach.ts`, `web-app/content/wiki/` directory tree, `web-app/content/wiki/.ingest-manifest.json`. Reads `coach/` read-only. Does NOT modify `coach/` itself.

**v1 scope clarification**: The ingestion script treats `web-app/content/wiki/` as fully owned by the script. Re-running deletes outputs for source files that no longer exist in `coach/`. The "preserves user-local content" framing in the original spec was hand-wavy and over-promised; in v1 the output dir is a mirror only, and any human-side customizations belong in a separate directory (not implemented in v1 but reserved for v2 if needed). Document this in the script's header comment so future agents don't expect mixed-mode behavior.

## Analysis

The ingestion script came out cleanly because R-018 had finalized the spec with no ambiguity to negotiate. The 6 normalization rules translated almost line-for-line into TypeScript. Three implementation decisions worth noting:

1. **Frontmatter rebuild strategy: pass-through with selective rewrite.** Rather than parse-and-reserialize the entire frontmatter (which risks reordering keys, changing quoting style, dropping comments), the script preserves frontmatter lines verbatim and only rewrites lines beginning with `related:`. This avoids subtle whitespace diffs in the output and keeps the idempotency hash stable across re-runs even if the YAML-lite parser ever gets stricter.

2. **Inlined derived-file regeneration instead of shelling out to `rebuild-index.sh`.** Per the R-018 + audit guidance, the script regenerates `_index.md` and `_backlinks.json` from the normalized output (not from `coach/`). I chose to reimplement the algorithm inline (≈50 LOC) rather than parameterize the Bash script. Reasons: (a) the Bash script hard-codes `COACH_DIR` and would have needed a TARGET_DIR parameter; (b) inlining keeps the ingestion script self-contained — no Python dependency, no shell-out, no working-directory subtleties; (c) the algorithm is trivial and unlikely to drift. Trade-off: two implementations of the same logic. Mitigation: both produce text/JSON outputs that can be diffed if drift is suspected.

3. **Integrity check is enforced only for full-corpus runs.** Subset runs (`--limit N` or `--only CATEGORY`) skip the threshold check because cross-category targets are excluded by construction. The script prints a note in subset mode so the operator knows the integrity number is informational, not load-bearing.

R-018's prediction that the corpus would clear 97% on its own held: 98.63% body, 98.46% related, both above the threshold by ≈1.5 points. The case-fold rule auto-fixed exactly the predicted set (verified by spot-checking `love-as-deep-welcoming.md` and `four-faces-of-love.md`).

The 466 remaining broken body links and 231 broken `related:` entries are recorded in the manifest for G-007's docket — already noted as G-007 ownership in R-018.
