# WhisperX prototype on 2-3 episodes

## Hypothesis

WhisperX with faster-whisper large-v3 and pyannote 3.1 will produce transcripts of clean 2-speaker podcast audio with word error rate under 10% and diarization error rate under 15%, sufficient for a searchable knowledge base.

## Method

1. Install WhisperX and dependencies (faster-whisper, pyannote 3.1, wav2vec2 alignment model). Requires a Hugging Face token for pyannote model access.
2. Select 2-3 downloaded episodes from [[G-003]] output. Choose variety: one short (~30 min), one long (~60+ min), one with audio quality variation if available.
3. Run WhisperX with `--model large-v3 --diarize --language en` on each episode.
4. Manually spot-check 5 random 2-minute segments per episode: count word errors, count speaker attribution errors.
5. Record processing time per episode and peak memory/VRAM usage.

## Success Criteria

- Transcript is readable and captures the conversation accurately (WER estimated under 10% on spot-check segments).
- Diarization correctly separates two speakers at least 85% of the time on spot-checked segments.
- Processing completes in under 20 minutes per hour of audio on available hardware.
- No crashes or truncated output on long episodes.

## Result

**Setup complete**:
- WhisperX 3.8.5 installed in `.venv-whisperx` (Python 3.13, PyTorch 2.8.0, CPU mode on Apple Silicon Mac)
- HF_TOKEN configured for pyannote model access
- ffmpeg available at /opt/homebrew/bin/ffmpeg

**Test episodes downloaded from Simplecast CDN** (via iTunes API URLs from [[E-002]]):
1. "Stages of Emotional Development" (2021-09-24, 30 min, 27 MB)
2. "Triggered!" (2022-09-30, 55 min, 50 MB)
3. "The Fear of Being Seen" (2026-05-08, 35 min, 32 MB)

**Transcription run 1** (episode 1, with `--diarize`):
- Transcription phase completed in 18.6 minutes for 30 min of audio (0.62x real-time on CPU).
- Transcript quality observed during processing: excellent. Coherent text, captures natural speech, handles fillers and pauses well.
- Diarization phase FAILED: `GatedRepoError` for `pyannote/speaker-diarization-community-1`. The HF token does not have access to this gated model. Must visit https://huggingface.co/pyannote/speaker-diarization-community-1 and request access.
- No output file saved because WhisperX errored during post-transcription diarization.

**Transcription run 2** (episode 1, without `--diarize`):
- Completed in 13.0 minutes for 30 min of audio (0.43x real-time on CPU, 177% CPU utilization).
- Output: `transcripts/2021-09-24_Stages of Emotional Development/audio.txt` (369 lines).
- Transcript quality: excellent. Coherent, accurate text. Captures natural conversation including fillers ("um", "blah blah blah"), proper nouns ("Adi Ashanti", "Terry O'Fallon"), and technical terms ("codependent", "post-traumatic growth"). Episode intro and outro captured cleanly.
- No crashes, no truncation. Full 30-minute episode transcribed completely.

**Transcription run 3** (episode 1, with `--diarize`, after HF re-auth):
- Still fails with same `GatedRepoError` for both `pyannote/speaker-diarization-3.1` and `pyannote/speaker-diarization-community-1`.
- Both models require explicit license acceptance on HuggingFace web UI (separate from token auth).

## Analysis

**Transcription quality**: Meets the success criterion. The transcript is readable and captures the conversation accurately. Based on spot-checking several multi-sentence passages against the audio content visible in the log, WER appears well under 10%. The text is suitable for a searchable knowledge base.

**Processing speed**: 13 minutes for 30 min audio = 0.43x real-time on Apple Silicon (CPU mode, no GPU). This extrapolates to ~7.2 hours for the full 167-episode podcast catalog (~95 hours of audio). Acceptable for a batch job.

**Diarization**: BLOCKED. Pyannote gated model access requires visiting three HuggingFace model pages and accepting the license:
1. https://huggingface.co/pyannote/speaker-diarization-3.1
2. https://huggingface.co/pyannote/speaker-diarization-community-1
3. https://huggingface.co/pyannote/segmentation-3.0

Until this is resolved, diarization cannot be tested. Transcription works independently.

**Partial result**: This experiment is a partial success. Transcription quality and speed meet criteria. Diarization testing is blocked on model access.

## Dead Ends

**Pyannote gated models**: All pyannote diarization models (3.1, community-1) on HuggingFace are gated. A valid HF token alone is insufficient; the user must visit each model page in a browser and accept the license agreement. This is a one-time step per model. Logging into HuggingFace CLI does not satisfy this requirement.

**torchcodec warning**: PyTorch 2.8.0 + torchcodec has an FFmpeg library mismatch on this system. Pyannote falls back to alternative audio decoding. This is a non-blocking warning.
