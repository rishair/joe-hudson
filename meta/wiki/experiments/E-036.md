# Coach v4: graph-walk retrieval over related: links in coach/

## Hypothesis

The `coach/` compendium is structured as a hand-curated knowledge graph: every file has a `related:` array with 5-10 expert-chosen semantic edges, and `meta/wiki/_backlinks.json` (5,624 lines) encodes the inverse graph. Walking this graph from a seed file discovered in the client message (e.g., `concerns/i-cant-apologize.md`) produces a "neighborhood" of relationally-adjacent files that encode the exact navigation Joe would do mentally. Graph-walk retrieval outperforms E-033 (embedding) by mean delta >= 0.2 on at least 2 of 6 G-008 dimensions, especially Methodology Fidelity (D4) where the hand-curated edges should map closely to which moves follow which reads.

The audit's central insight: embedding similarity retrieves topically-adjacent files; graph-walk retrieves relationally-adjacent files. For a coaching system whose central skill is "where does Joe go from here," the latter matches the task shape.

## Method

1. Read E-031's coach-app/ scaffolding. Reuse the v1 system prompt (held constant across all retrieval experiments). R-012 has shipped (2026-05-22) and provides the design spec below.
2. Build the seed-detection step (per R-012):
   - On each turn, use Haiku 4.5 with the full `coach/_index.md` (684 lines) inlined as input — every file's aliases are listed inline, making this essentially a lexical-lookup task that Haiku handles cheaply (~$0.001-0.003 per call).
   - Output: 1-3 seed file IDs, constrained to `concerns/`, `patterns/`, or `reads/` (those are the "where the client is" surface; everything else is "where Joe goes from here" which the walk will reach).
   - Fallback when seed-detection returns nothing: dense-similarity-over-aliases seed-finder (use the same bge-small-en-v1.5 index E-033 builds; reuse, don't duplicate).
3. Implement graph-walk at `coach-app/retrieval/graph-walk.ts` (per R-012):
   - Parse `related:` arrays from each seed's frontmatter; the file body is the source of truth. Use `coach/_backlinks.json` (652 nodes, 4,598 edges, mean indegree 7, max indegree 134) for inverse lookups.
   - Walk to depth 2, deduplicating.
   - Score nodes: 1.0 for direct seed neighbors, 0.5 for 2-hop, +0.3 bonus for nodes reached from multiple seeds (centrality).
   - **Hub-dampen** (R-012 critical design choice): multiply each node's score by `1 / log(1 + indegree)` from `_backlinks.json`. Without this dampening, hubs like `feeling-the-unfelt-emotion` (indegree 134), `vulnerability` (54), `shame-drives-behavior` (49) would always retrieve and dilute attention from situationally-specific files.
   - Cap retrieval bundle at top 5-7 files by score (default K=5).
   - Return full markdown bodies (not summaries — R-012's injection format).
4. Inline retrieved markdown into the coach's context under "RETRIEVED CONTEXT (graph-walk neighborhood)".
5. Create `coach-app/configs/v4-graph-walk.yaml` with the same v1 system prompt (plus the retrieval-aware language added per audit Important #5) and the graph-walk retriever wired in.
6. Smoke on 2 profiles. Validate: seeds are sensible, walked neighborhood looks topologically coherent, coach response uses retrieved content.
7. Run full 15-profile eval. Estimated cost: $8-10 (Haiku seed-detection per turn is cheap, no embedding model needed, walk is free).
8. Render scorecard via E-030. Compare to E-031 and to E-033.
9. Log: which seed files were chosen per profile? Did the walk consistently surface high-quality neighbors? Were there profiles where graph-walk found files embeddings missed (or vice versa)?
10. Write Result with delta vs E-031 AND delta vs E-033 (this is the head-to-head with the most architecturally-similar approach), Analysis on whether the graph structure pays off.

Implementation ramp: smoke on 2 profiles before full run.

## Success Criteria

- Graph-walk retriever runs end-to-end: seed selection, walk, scoring (with hub-dampen), return
- Per-turn telemetry logs the seeds and the walked neighborhood (file IDs + scores + which seed each came from)
- **expected_territory pre-screen passes (per R-012)**: precision/recall of retrieved file IDs against each profile's `expected_territory` block in smoke phase; if recall < 40% averaged across smoke profiles, debug seed-detection or walk depth before paying for full eval. R-012 predicts graph-walk will beat hybrid embedding on expected_territory recall specifically because the curated edges encode the same expert judgment as `expected_territory`.
- Full G-008 eval produces a scorecard
- Mean delta vs E-031 reported per dimension
- Mean delta vs E-033 reported per dimension (this is the architectural comparison: graph-walk vs hybrid embedding)
- **Specifically: D4 (Methodology Fidelity) ≥ 2.50** (baseline E-024=2.07; R-012 predicts graph-walk is the architecturally-strongest D4 mover)
- Total experiment spend logged and under $11
- Verdict: did graph-walk beat E-033 on any dimensions? Was the graph structure the right primary retrieval substrate?

Failure mode: seed-detection misfires (Haiku picks irrelevant `coach/` files), or 2-hop walks return overly large neighborhoods that dilute coach attention. Both are tunable; document specific failures with examples.

## Analysis

R-012's primary prediction held: graph-walk outperforms embedding on D3 Stance and D4 Methodology Fidelity, with the curated `related:` edges encoding the exact navigation pattern an AoA coach would mentally take. The hub-dampening design choice was load-bearing — without `1/log(1+indegree)` scaling, hubs like `feeling-the-unfelt-emotion` (indegree 134) would have dominated every retrieval and diluted situational specificity.

R-012's secondary prediction (D3 Stance regresses under retrieval) was falsified for graph-walk specifically. E-036's D3 score of 2.67 exceeds E-031 v1's 2.60. The likely reason: graph-walk's tighter, less-noisy retrieval bundle does not interfere with the coach's stance the way embedding's broader topical matches do (E-033 D3 = 2.53, a regression vs v1). This suggests retrieval-induced stance interference is a function of bundle relevance, not retrieval per se.

E-036 wins the four-way architectural comparison (E-031/E-032/E-033/E-036) on overall mean. However, the delta vs E-033 (+0.04 overall, +0.13 on D3, +0.10 on D6, -0.13 on D1) does NOT meet E-034's "clear winner" criterion (mean delta >= 0.2 over next-best on at least 3 dimensions, no regression worse than 0.1). E-036 vs E-033 has zero dimensions at +0.2 and one regression at -0.13. Per E-034, this is the per-dimension-split scenario that **mandates E-037 hybrid** rather than going directly to E-035.

Crisis-coercion-001 remains the unsolved profile across all retrieval architectures. The somatic-shutdown read requires both (a) detecting dissociation as a state change distinct from the prior emotional content (a D1 read), and (b) deploying somatic grounding / threshold calibration rather than continued emotion-feeling moves (a D2 move + D4 fidelity). Graph-walk surfaces some of the right files but the v1 prompt does not instruct the coach to pivot at dissociation. This is the highest-value target for E-035 prompt iteration regardless of which retrieval architecture is the final substrate.

Cost: E-036 came in at $11.09 vs $11 ceiling. The graph-walk retrieval injection inflates coach input tokens but the Haiku seed-detection cost is negligible. Net effect is close to E-033 ($11.74) and about 2.6x E-031 v1 ($4.23). The audit-flagged "retrieval inflates context, shreds cache" effect is real but smaller for graph-walk than for embedding's hybrid alias-BM25 + dense bundle.
