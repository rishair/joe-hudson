# Deploy dashboard via G-012 PaaS at status.play.ris.hair with Cloudflare Access auth

## Hypothesis

The dashboard from E-050, plus a small `.local-dev.yaml`, can be deployed via G-012's `paas` CLI to `status.play.ris.hair` and reached from any browser after Cloudflare Access SSO. The whole flow takes one command (`paas register .`) once the project's YAML is in place.

This experiment closes G-013 and also closes G-012's "first dogfood project" exit criterion (via [[E-049]]).

## Method

1. Confirm E-050 (dashboard build) is complete with a working `bun run start`.
2. Confirm G-012's E-048 (paas CLI) is complete.
3. Author the dashboard project's `.local-dev.yaml`:
   ```yaml
   subdomain: status
   port: 4001  # or whatever is free
   command: bun run start
   mode: built
   working_dir: /home/claude/joe-hudson/status-dashboard
   env_file: .env
   restart: always
   ```
4. Run `paas register /home/claude/joe-hudson/status-dashboard`.
5. End-to-end verify (per [[E-049]]'s checklist): URL reachable, CF Access auth works, dashboard renders real data, restart resilience.
6. From mobile browser (real-world test): does the dashboard render readably on a small screen? If not, log as a follow-up nice-to-have (mobile-responsive layout) — don't block on it for v1.
7. Update G-013's exit criteria: tick everything that's now true.

## Success Criteria

- `https://status.play.ris.hair` resolves through CF Access
- After SSO, the dashboard renders with all 4 must-have sections populated
- G-013's "deployed via G-012's PaaS" exit criterion is met
- G-012's "one project registered and reachable" exit criterion is met (this is the same flow as [[E-049]] — they share the verification)
- User can confirm subjectively: "I can glance at this from my phone and know what's happening"
- Any deploy-time friction surfaces a follow-up (not a fix in this experiment)

## Blast Radius

- Adds the dashboard as a registered paas project (systemd unit, tunnel ingress)
- Reachable at the public CF Access-gated URL
- Does NOT modify the dashboard source — only registers it
