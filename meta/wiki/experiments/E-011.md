# Full podcast episode audio download

## Hypothesis

Downloading all 167 podcast episodes from the iTunes API URLs (stored in `meta/raw/itunes-episode-list.tsv`) via curl will produce a complete set of MP3 files. Based on the successful 3-episode test in [[E-009]], all Simplecast CDN URLs should resolve.

## Method

1. Read `meta/raw/itunes-episode-list.tsv` (167 episodes with date, title, duration_ms, audio URL).
2. For each episode, create a directory and download the MP3:
   ```
   tail -n +2 meta/raw/itunes-episode-list.tsv | while IFS=$'\t' read -r date title duration_ms url; do
     safetitle=$(echo "$title" | sed 's/[\/:]/-/g' | cut -c1-80)
     dir="transcripts/${date}_${safetitle}"
     mkdir -p "$dir"
     if [ -f "$dir/$safetitle.mp3" ]; then
       echo "SKIP: $dir"
     else
       echo -n "DL: $dir ... "
       curl -sL -o "$dir/$safetitle.mp3" -w "HTTP %{http_code}, %{size_download} bytes\n" "$url"
       sleep 1
     fi
   done 2>&1 | tee meta/raw/podcast-download.log
   ```
3. After completion, count downloaded files and compare against 167 expected.
4. Spot-check 5 random files with `file` command.

## Success Criteria

- 165+ MP3 files downloaded (allowing 1-2 potential failures).
- All HTTP responses are 200.
- All files are valid MPEG audio.
- Total download size is in the expected range (~7-8 GB).
