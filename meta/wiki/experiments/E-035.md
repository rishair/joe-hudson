# Walker keep-threshold tuning (REFRAMED from "iterate system prompt v4-v7")

## Reframe note (2026-05-22)

The original E-035 spec was "iterate system prompt v4 → v7 on winning retrieval architecture." After E-038 (Sonnet walker) landed, that diagnosis was wrong. E-038 telemetry showed the Sonnet walker keeps **31% of files it reads** (vs Haiku's 43%) with a 27% smaller bundle (1.86 files vs 2.56). D6 dropped to 2.40 (from E-037's 2.60) precisely because the coach has less depth-enabling material per turn. The catastrophic edge-not-anxious-001 regression (2.33 → 1.33) is a coach-pursuing-fear-frame failure — the right files (`disassociation-as-numbness-layer`, `the-happiness-filter`, etc.) plausibly weren't surfaced to let the coach stay at texture instead of escalating to interpretation.

**The bottleneck is walker selectivity, not the system prompt.** The walker is being TOO discriminating: it asks "is this file what Joe would draw on for this exact turn?" and rejects files that could give the coach material to go deeper. The fix lives in walker tuning, not in iterating the v1 prompt that has already been shown to be a strong general-purpose foundation by every retrieval experiment that held it constant (E-031, E-033, E-036, E-037, E-038).

So E-035 is reframed: tune the walker's keep-decision language to value depth-enabling files, target acceptance rate ~38-42% (between E-037 Haiku's 43% and E-038 Sonnet's 31%), and see if D6 recovers without D1 regressing.

## Hypothesis

The Sonnet walker's selectivity is good for D1 (perceptual accuracy — fewer noisy files = cleaner read) but bad for D6 (depth — fewer files = less material for the coach to compose with). Adding language to the walker's keep-decision criteria that explicitly values "files that could give the coach material to go deeper, not just files that name the central concept" will raise the acceptance rate from 31% toward 38-42%, recover D6 toward E-037's 2.60, and rescue the edge-not-anxious-001 profile (because depth-enabling files like `disassociation-as-numbness-layer` will be kept where previously they would have been rejected as "too tangential").

## Method

1. **Read `coach-app/retrieval/guided-walk.ts`** to find the walker's keep-decision prompt (`WALKER_SYSTEM` constant + the `add_to_bundle` field description in `STEP_DECISION_SCHEMA`).
2. **Modify the walker prompt** to add depth-enabling-file guidance. Concrete approach (a) from the spec: add language in the DECISION A section valuing depth-enabling files. Approach (b) — lowering an explicit threshold — is not available here; the walker's decision is binary via the structured-output `add_to_bundle` boolean, so prompt language is the only lever.
3. **Create `coach-app/configs/v6-walker-tuned.yaml`** — identical to v5b except for the walker-prompt swap. Use a `walker_variant` flag in the config to keep v5b reproducible (the change to `guided-walk.ts` adds a "depth-aware" branch selected by config, not a wholesale replacement).
4. **Smoke on 2 profiles**: `edge-not-anxious-001` (the specific failure target) and `client-happy-am-i-selfish-001` (same happy-path used in v5b's smoke, allows direct comparison). Smoke cost ~$2.
5. **Compare smoke to v5b smoke**: does the targeted profile recover? Does telemetry show acceptance rate moved into the 38-42% target band? Did the happy-path baseline regress?
6. **Decision gate**:
   - If smoke shows D6 recovery (≥2.50) without D1 regression (≥2.70 on the happy-path baseline), promote to full 15-profile eval (~$15-18).
   - If smoke shows no recovery, **STOP** — document the dead end and do NOT burn the full-eval budget on a flat result.
7. **Render via E-030**. Compare to E-038 (v5b) and E-037 (v5a).
8. **Update G-009 iteration log** with the v6 row and a decision log entry.

### Budget guard (tightened from original $15 cap)

Cap total spend at **$5**. G-009 is already $1 over its $60 cap; the parallel R-014 audit is +$0-2; this experiment's $5 ceiling lands G-009 at $66-68 worst case. If smoke ($2) shows no movement, the full eval ($15-18) is NOT spent. Hard stop at $5.

### Implementation ramp

Smoke-first. No full eval without smoke evidence of movement. The "two diagnoses" rule from the original spec doesn't apply at this scope — we have one clearly-motivated change to test.

The right files for this profile (`disassociation-as-numbness-layer`, `chin-up-as-emotion-block`, `intellectual-process-with-no-somatic-component`, `i-cant-feel-my-emotions`) **did surface in the bundle on turns 2, 4, 8**. The walker tuning worked at the retrieval layer.

But the coach STILL escalated past the numbness texture. Coach turn 8 said: "that's not nothing. That's a specific kind of gap" — interpreting the absence of somatic content as a finding to investigate, instead of staying with "fine" until "fine" has a quality (the golden path). By turn 14: "you said 'more of me in it' and then immediately hedged — 'I wouldn't call that fear' — what were you about to say?" — explicitly pushing toward the fear-frame the client just declined. Judge scored D5 anti-pattern-avoidance=1 because the coach evangelized the work ("that's actually useful information") and pursued the fear-frame past the client's no.

**The bottleneck shifted, but not all the way to the system prompt.** Two compounding causes for the persistent edge-not-anxious-001 failure:

1. **Coach interpretation reflex.** Even with the right depth-enabling files in the bundle, Sonnet-as-coach defaults to interpreting/labeling rather than staying at texture. The v1 prompt's "don't lecture" doesn't bind tightly enough on the move "stay with 'fine' until 'fine' has a quality."
2. **Seed-detection failures on 2-3 turns.** Turns 6, 10, 14 produced empty/wrong seed sets, so retrieval contributed nothing to those turns. The walker variant can only help on turns where the seed detector lands somewhere usable.

These are both orthogonal to walker keep-threshold tuning. They are real next-experiment targets (prompt iteration AND seed-detector tuning) but out of scope for E-035 reframed.

## Analysis

The hypothesis "walker selectivity is the bottleneck for D6 / edge-not-anxious-001" is **partially confirmed and partially falsified**.

Confirmed: walker selectivity *was* a contributor. The depth-aware variant produced exactly the mechanical change it was designed to produce (acceptance rate 31% → 44-45%, bundle size 1.86 → ~2.70). On turns where seed detection succeeded, the depth-enabling files like `disassociation-as-numbness-layer` and `chin-up-as-emotion-block` *did* get surfaced where v5b would have skipped them. Edge-not-anxious-001 mean went up +0.17 (1.33 → 1.50) and D2 intervention-quality went up +1.

Falsified: walker selectivity was NOT the dominant cause of the catastrophic 2.33 → 1.33 v5b regression. Even with the right files in the bundle, the Sonnet coach still pushed past numbness into interpretation and fear-framing. D5 (anti-pattern-avoidance) stayed at 1 and D6 (effectiveness-depth) stayed at 1. Walker tuning provides the coach better raw material, but a coach that defaults to interpretation will interpret with whatever material it has.

The mechanistic finding from telemetry is clean: **the variant change is doing its job at the retrieval layer, but the coach is no longer the constrained component.** Two non-walker bottlenecks now stand out as the next-experiment targets:

1. **System prompt language on "stay at texture, don't interpret."** The v1 prompt's "Don't lecture / Don't evangelize" doesn't bind on the specific move "when the client cannot find an emotion, stay at the body's reading until the body's reading has a quality — do NOT name a finding from the absence." This is a precise instruction the v1 prompt does not contain. A v7 prompt iteration targeting this would be a clean follow-on experiment.
2. **Seed detection failures on ~30% of turns** (3 of 7 retrieval calls on edge-not-anxious returned 0 seeds or wrong seeds). Tuning the seed-detection prompt (currently Haiku 4.5 reading `coach/_index.md`) to be more generous when the client is presenting as "fine / no problem" is a separate but compatible next-experiment.

For G-009's exit-criteria, this experiment **does not move the needle on the goal-page exit bar**: edge-not-anxious-001 is one of 15 profiles, and a +0.17 on it does not change the aggregate that already met the conditional ≥0.20 bar in E-038 (all 6 dims improve vs E-024 by ≥0.07). The G-009 exit-criteria question (≥4 dims with conditional delta vs E-024, AP triggers <50% of E-024) is already nominally satisfied by E-038's full-eval scorecard. What E-035 reframed was supposed to add was the **strict-success-criterion** clearance on top of that — which neither the original prompt-iteration plan nor this walker-tuning attempt has cleared.

## Dead Ends

- **Walker keep-threshold tuning is not the primary lever for edge-not-anxious-001.** The hypothesis that bundling more depth-enabling files would let the coach stay at texture is mechanistically reasonable but empirically does not move D6 or D5 on this profile when the coach is Sonnet running v1 prompt. The retrieval-layer fix is necessary-but-not-sufficient; the prompt-layer fix on "do not interpret absence as a finding" remains untested.
- **Approach (b) — lowering an explicit threshold — was not available.** The walker's add_to_bundle decision is a boolean from a structured-output schema; there is no numeric threshold to dial. Prompt language was the only lever.
- **The 38-42% acceptance target was overshot to 44-45%.** Not a problem — the direction is correct and the bundle still well below k_max — but if a future iteration wants to tune to a specific narrow band, the language could be slightly less generous than the current `depth-aware` variant.

## Next Steps

Two specific next-experiment ideas a future agent should consider (not pursued here under the $5 cap):

1. **v7 system prompt targeting the "absence is information; stay at texture" move.** Add language to v1.md's methodology section that explicitly addresses the client-cannot-find-an-emotion case: "When the client says 'fine' or 'nothing,' do not interpret the absence as a finding to investigate. Ask what 'fine' actually feels like, and stay with the texture until the texture has a quality." This is a small, targeted prompt edit ($5-9 to test with v6 walker-tuned config). Likely the highest-leverage next move for closing the edge-not-anxious-001 gap.
2. **Tune the Haiku seed-detection prompt to surface seeds even when the client presents as 'no problem here.'** On 3 of 7 turns the seed detector returned empty/wrong seeds, meaning the coach got NO retrieval on those turns. The seed-detector prompt currently selects from `concerns/patterns/reads/` — adding language like "if the client is denying the concern itself, the concern is still present; surface concerns/ files that match the denied territory" could improve recall on the denial-of-concern edge case.

Neither is in scope for E-035 as reframed; both are good candidates for a follow-on if G-009 gets a second budget tranche.

If neither follow-on is funded, the G-009 picture is: E-038 Sonnet walker is the deliverable retrieval architecture; v1 system prompt is the deliverable prompt; depth-aware walker variant exists as a code path behind a config flag for future use; edge-not-anxious-001 is the named open gap that future prompt iteration would target.
