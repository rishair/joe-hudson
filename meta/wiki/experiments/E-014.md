# AssemblyAI bulk transcription with diarization

## Hypothesis

AssemblyAI's pre-recorded transcription API with `speaker_labels: true` will transcribe all downloaded podcast and YouTube audio files with accurate speaker diarization at a cost of ~$25 for ~150 hours of audio, completing within a few hours of wall time.

## Method

1. **Prerequisite**: `ASSEMBLYAI_API_KEY` must be in `.env`. Install SDK: `pip install assemblyai` in `.venv-whisperx` or a new venv.
2. **Prototype on 3 test episodes** (same ones from [[E-005]]): Submit via the Python SDK with `speech_models=["universal-3-pro", "universal-2"]` and `speaker_labels=True`. Verify transcript quality and diarization accuracy. Confirm cost per episode.
3. **Build bulk submission script** (`scripts/transcribe_assemblyai.py`):
   - Walk `transcripts/` directories, find all `audio.mp3` files.
   - For each, upload via `aai.Transcriber().transcribe(local_path)` with diarization enabled.
   - Save result as `transcript.md` in the same directory with speaker labels formatted as `**Speaker A:** text`.
   - Use `--download-archive`-style tracking (a `transcribed.txt` log) to skip already-processed files on re-runs.
   - Submit in batches (10-20 concurrent) to avoid overwhelming the API. The SDK handles polling internally.
4. **Run on full catalog**: All podcast episodes and YouTube audio files.
5. **Verify**: Spot-check 5 random transcripts for quality. Count total files with `transcript.md`. Compare against catalog counts.

## Success Criteria

- Prototype: 3 test episodes produce readable transcripts with speaker labels. Cost is under $0.25/episode.
- Full run: 90%+ of downloaded audio files have a `transcript.md` with speaker diarization.
- Total API cost under $50.
- Wall time under 6 hours for the full catalog.

## Result

**Prototype** (3 test episodes):
- All 3 transcribed successfully in 31-53 seconds each via SDK.
- Speaker diarization correctly separated speakers (2 speakers in 2-person episodes, 3 in one multi-guest episode).
- Transcript quality: excellent. Proper nouns, technical terms, and natural speech patterns all captured accurately.

**Full run** (two passes + backfill):
- Pass 1 (old script): 410 transcripts in ~30 minutes. No utterances.json (old script version).
- Pass 2 (updated script): 65 remaining files, with both transcript.md and utterances.json.
- Backfill pass: Re-submitted 410 episodes to get utterances.json. Completed in ~40 minutes.
- **Final count: 475 transcript.md files and 475 utterances.json files** out of 479 audio files (99.2% coverage).
- Total wall time: ~2 hours across all passes (including backfill).

**Output per episode**:
- `transcript.md`: Speaker-labeled markdown (`**Speaker A:** text`, `**Speaker B:** text`)
- `utterances.json`: Structured data with speaker label, text, start (ms), and end (ms) per utterance. Needed for voiceprint-based speaker relabeling ([[E-015]]).

## Analysis

All success criteria met:
- Prototype: readable transcripts with speaker labels, well under $0.25/episode.
- Full run: 475/479 = 99.2% coverage (exceeds 90% target). The 4 missing files may be empty or corrupt audio.
- Wall time: ~2 hours (well under 6-hour target).
- API cost: estimated $15-25 based on ~150 hours of audio at $0.17/hr (exact cost available in AssemblyAI dashboard).

Speaker diarization produces labels like "Speaker A", "Speaker B", not names. The voiceprint relabeling experiment ([[E-015]]) will use the utterances.json timestamps to match Joe Hudson's voice and relabel.

None. AssemblyAI worked on first try for all tested files.
