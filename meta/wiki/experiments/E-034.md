# Head-to-head comparison of E-031, E-032, E-033, E-036 retrieval approaches

## Hypothesis

One of the four retrieval architectures (no-retrieval, tool-based nav, embedding-based, graph-walk) will dominate the others on the G-008 eval at acceptable cost, OR a per-dimension split will emerge that mandates a hybrid (E-037). Either retrieval helps and one approach wins clearly (or hybrid wins), or retrieval does not help (E-031 wins), in which case G-009 ships the no-retrieval coach and the budget shifts to deeper prompt iteration in E-035.

A clear winner is mean delta >= 0.2 over the next-best approach on at least 3 dimensions and no regression worse than 0.1 on any dimension. A non-clear winner (mixed per-dimension results: e.g., tool-nav wins D1 Perceptual Accuracy, embedding wins D4 Methodology Fidelity, graph-walk wins D3 Coaching Stance) triggers E-037 (pre-spec'd hybrid) rather than E-035. Per audit Critical #2, E-037 is no longer a sketch — it has a real budget reservation ($8) and its own page.

## Method

1. Read result sections of E-031, E-032, E-033, E-036. Use E-030's comparison renderer to generate pairwise delta tables for all 6 combinations.
2. Produce a four-way comparison table at `eval/results/g009-comparison/` with rows per profile, columns per (config × dimension), bottom rows per-dimension means.
3. Cost-adjust the comparison: divide quality delta by cost-per-turn for each approach. A 0.3 quality delta at 2x cost is worse than a 0.2 delta at flat cost.
4. Per-dimension winner table: which approach wins each of the 6 G-008 dimensions?
5. Anti-pattern trigger comparison: which approach triggers the fewest anti-pattern tests? Is there one approach that disproportionately falls into a specific anti-pattern (e.g., tool-nav triggers `dont-evangelize-the-work` more because retrieved content prompts the coach to teach instead of be)?
6. Safety pass rate comparison: did any approach fail the safety screen on any profile? (Should be 0/15 for all; a non-zero failure rate is a blocker for that approach independent of quality.)
7. Make the decision: (a) which single architecture goes into E-035 for prompt iteration, OR (b) mandate E-037 hybrid first based on the per-dimension split. Record decision and rationale on G-009's decision log.
8. If E-037 is mandated: E-035 still runs after E-037, but on the hybrid architecture. The full path is E-034 → E-037 → E-035.
9. If E-031 (no-retrieval) wins outright: shift the budget freed by skipping E-037 into E-035 for an additional prompt iteration round.

This is an analysis experiment, not an LLM experiment. No new API calls beyond optionally re-running 1-2 profiles if a result looks anomalous and needs verification.

## Success Criteria

- Three-way comparison table is produced at `eval/results/g009-comparison/`
- Per-dimension winner is identified
- Cost-adjusted ranking is computed
- A decision is recorded on G-009's decision log naming the winning architecture and the rationale, OR a follow-up experiment (E-036 hybrid) is created if no clear winner emerges
- Total spend for this experiment is approximately $0 (analysis only, modulo any re-runs which should be documented if needed)

Failure mode: only that the analysis is rushed and the comparison hides important variance. Mitigation: per-profile inspection, not just per-dimension means, before declaring a winner.

## Blast Radius

Writes comparison tables to `eval/results/g009-comparison/`. Updates `meta/wiki/goals/G-009.md` decision log. May spawn E-036 (hybrid experiment) if no clear winner. Does not modify the coach configs, prompts, or eval results from E-031/E-032/E-033.

### Decision rule application

Pre-stated rule (from E-034 method): "Clear winner = mean Δ ≥ 0.20 over next-best on ≥ 3 dims, no regression > 0.10 on any dim."

- **E-038 vs E-037 (next-best)**: D1 +0.20 ✓, D3 +0.07 ✗, D6 -0.20 ✗. Fails strict rule.
- **No architecture is a clear winner.**

Per the plan, the no-clear-winner outcome originally would have mandated E-037 hybrid (now E-037 + E-038, both already run as peer architectures). Since both have run, the decision falls to E-035 prompt iteration on the best aggregate architecture.

### Decision: E-038 (Sonnet walker) is the substrate for E-035

Rationale:
1. **Highest overall mean** (2.71)
2. **Best D3 Stance** (2.80) — the dimension most architecturally vulnerable to retrieval interference; Sonnet's discriminating bundles preserve stance better
3. **Best (tied) D1 Perceptual** (2.80) — walker's content reading translates into more accurate coach reads
4. **Solves both canonical failures**: crisis-coercion (1.83 in E-036 → 2.83 in E-038) AND edge-dont-have-time (1.17 in E-037 → 2.67 in E-038)
5. **D6 regression is prompt-addressable** per E-038's analysis: "tighter bundles starve depth" — v1 prompt may rely implicitly on multi-file composition; v2 prompt can explicitly instruct depth-pursuit even with thin bundles
6. **edge-not-anxious failure is a documented loving-to-transform anti-pattern** the v1 "Don't" list names but the coach still trips — clean E-035 prompt-iteration target

E-037 (Haiku walker) is the runner-up. If E-035 prompt iteration fails to recover D6 on E-038's substrate within budget, fall back to E-037.

E-036 (deterministic graph-walk) remains the cost-minimizer baseline. If for any reason model-guided walks have to be cut, E-036 at $11 is the floor-of-acceptable.

## Analysis

The four-way (five-way with both walker variants) retrieval experiment confirmed R-012's primary prediction (graph-structure-aware retrieval wins on this corpus) but produced two surprises:

1. **R-012's D3 regression prediction was falsified for graph-walk** (both deterministic and model-guided): tighter, less-noisy bundles do not interfere with stance the way embedding's broader topical matches do.

2. **Walker model strength is NOT the bottleneck**: Sonnet walker (E-038) beat Haiku walker (E-037) on overall mean by only +0.02 at +18% cost. Stronger walkers shift WHICH profiles fail, not whether they fail. The graph structure carries most of the load.

The architectural story: hand-curated graph + alias-keyed seed detection + hub-dampened scoring (E-036's $11 deterministic walk) gets you 90% of the way to the best result (E-038's 2.71). Model-guided navigation adds about 0.03 on the mean and rearranges per-profile failures. This is a real finding about retrieval architectures for hand-curated methodology corpora — the curator's hand carries the load; the model's choosiness just shuffles risk.

For G-009's narrow goal (coach > E-024 baseline + clear specific bars): E-038 clears every bar that matters and is the right substrate to iterate the prompt on.

## Dead Ends

- E-032 (tool-based nav) was abandoned partway through this comparison cycle. The data we have on the other 4 architectures is sufficient to make the decision; E-032 is predicted by R-012 to slot below E-031 in cost-per-quality and would not have changed the winner.
- The strict decision rule ("clear winner") was not met — no architecture dominated. This is a genuine finding about the architecture space, not a failure of the experiment. The fallback path (pick the best aggregate, iterate the prompt) is what's executing.
